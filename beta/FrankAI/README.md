# FrankAI — учебный прототип собственной языковой модели

**Это не production и не настоящая LLM.** FrankAI — символьная
(char-level) рекуррентная нейросеть (vanilla RNN), написанная "с нуля"
на чистом NumPy и обучаемая обратным распространением ошибки во времени
(BPTT) на нескольких килобайтах текста. Цель проекта — увидеть и понять
весь путь языковой модели целиком: текст → числа → сеть → числа →
текст, — а не получить хороший или связный текст на выходе.

## Что здесь реально происходит

1. **Токенизация** (`frankai/tokenizer.py`) — каждая буква обучающего
   текста получает свой номер. Никакого BPE, никаких "под-слов" — самый
   простой вариант токенизации из возможных.
2. **Модель** (`frankai/model.py`) — одна рекуррентная сеть (`CharRNN`):
   на каждом шаге сеть смотрит на очередную букву и своё предыдущее
   "скрытое состояние" (память) и пытается угадать следующую букву.
   Прямой проход, обратный проход (BPTT), обрезка градиентов, шаг
   оптимизатора Adagrad — всё реализовано явно, без autograd-фреймворка,
   чтобы каждый шаг был виден в коде.
3. **Движок** (`frankai/engine.py`) — класс `FrankAI` связывает
   токенизатор и сеть в единый API: `train()`, `generate()`,
   `save()`/`load()`, `load_or_train()`.

## Единый интерфейс

```python
from frankai import FrankAI

# Если весов ещё нет — обучится на маленьком встроенном корпусе и
# сохранит результат; если веса уже есть — просто загрузит их.
engine = FrankAI.load_or_train(
    weights_path="weights/frankai_weights.npz",
    corpus_path="frankai/data/sample_corpus.txt",
)

text = engine.generate("Привет")
print(text)
```

`FrankAI` реализует абстрактный `Engine` (`frankai/engine.py`) —
единственное, что от него нужно другому коду, это метод
`generate(prompt) -> str`. Именно так его использует `Nex-pilot`
(`../Nex-pilot`), и по этому же принципу можно подставить любой другой
"движок" вместо FrankAI, не переписывая вызывающий код.

## Установка и запуск

Требуется Python 3.11+.

```sh
cd beta/FrankAI
python -m venv .venv
# Windows (PowerShell):
.venv\Scripts\Activate.ps1
# macOS/Linux:
source .venv/bin/activate

pip install -r requirements.txt
# или как editable-пакет (нужно для Nex-pilot): pip install -e .
```

Быстрая проверка:

```sh
python -c "
from frankai import FrankAI
e = FrankAI.load_or_train('weights/frankai_weights.npz', 'frankai/data/sample_corpus.txt')
print(e.generate('Привет'))
"
```

Первый запуск обучит модель (несколько секунд) и сохранит веса в
`weights/frankai_weights.npz` — все следующие запуски будут читать уже
готовые веса мгновенно. Чтобы обучить заново (например, с другими
параметрами), удалите файл весов или используйте `scripts/train.py`:

```sh
python scripts/train.py --iterations 15000 --hidden-size 128
```

## Тесты

```sh
pip install -r requirements-dev.txt
pip install -e .   # frankai должен быть импортируемым пакетом
pytest
```

`tests/test_frankai.py` — токенизатор (кодирование/декодирование,
неизвестные символы), прямой и обратный проход `CharRNN` (формы
тензоров, конечность и знак потери, обрезка градиентов, что шаг
оптимизатора действительно снижает потерю на повторяемом фрагменте),
сэмплирование (детерминизм при фиксированном `rng`, влияние температуры
на разнообразие выбора), обучение `FrankAI.train()` и, наконец,
сохранение/загрузка весов — включая проверку, что `load_or_train()` при
существующем файле весов ЗАГРУЖАЕТ их, а не обучает заново.

## Честная оценка качества (пожалуйста, прочитайте перед тем, как расстраиваться)

- Корпус — несколько килобайт текста. Настоящие языковые модели
  обучаются на терабайтах. Разница на много порядков, и это напрямую
  видно в качестве результата.
- Сеть маленькая (по умолчанию 128 "нейронов" в скрытом слое, один
  рекуррентный слой) и не использует ни трансформер, ни attention.
- **Результат генерации не будет связным осмысленным текстом.** Ожидайте
  что-то вроде правдоподобных по длине "слов", разумных пробелов и
  знаков препинания, иногда — узнаваемые обрывки фраз из обучающего
  корпуса. Не ожидайте ответов на вопросы, логики или фактов.
- Чем дольше обучение (больше `--iterations`), тем более "уверенно" сеть
  повторяет локальные закономерности корпуса — но она всё равно не
  начнёт рассуждать: у ванильной RNN такого размера просто нет для
  этого мощности, и это не баг, а суть архитектуры.
- Все эти упрощения — осознанные и сделаны специально, чтобы прототип
  оставался маленьким, быстрым и полностью прозрачным для чтения. Это
  и есть его учебная ценность: весь код умещается в три небольших
  файла и его реально прочитать целиком за один присест.

## Параметры генерации

- `max_new_tokens` — сколько новых символов сгенерировать (по умолчанию 200).
- `temperature` — "случайность" выбора следующего символа:
  меньше 1.0 — более предсказуемо и однообразно, больше 1.0 — более
  случайно (и чаще откровенно бессмысленно). По умолчанию 0.8.

## Структура файлов

```
beta/FrankAI/
├── frankai/
│   ├── __init__.py        # публичный интерфейс: from frankai import FrankAI
│   ├── tokenizer.py         # CharTokenizer — текст <-> числа
│   ├── model.py              # CharRNN — сеть: forward/backward (BPTT)/sample
│   ├── engine.py             # Engine (интерфейс) + FrankAI (train/generate/save/load)
│   └── data/
│       └── sample_corpus.txt  # маленький учебный корпус (самостоятельно написан)
├── scripts/
│   └── train.py              # CLI: обучить модель дольше/с другими параметрами
├── tests/
│   └── test_frankai.py       # юнит-тесты: forward/backward, sampling, save/load
├── weights/                  # сюда сохраняются обученные веса (.npz, не в git)
├── pyproject.toml
├── requirements.txt
├── requirements-dev.txt
├── .gitignore
└── README.md
```
