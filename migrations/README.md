# migrations/ — SQL-миграции (goose)

Формат имени: `NNNNN_описание.sql` (нумерация сквозная). Каждая миграция содержит секции `-- +goose Up` и `-- +goose Down`.

```sh
make migrate          # применить все новые миграции
make migrate-down     # откатить последнюю
goose -dir migrations create add_students sql   # создать новую
```

Строка подключения берётся из `NEX_DATABASE_URL` (см. `.env.example`).

Правила: миграция после попадания в main не редактируется — только новая поверх; DDL и большие бэкфиллы данных — раздельными миграциями; каждая таблица доменных данных обязана иметь `tenant_id` и RLS-политику (см. `internal/kernel/tenancy`).

`00001_init.sql` — черновик схемы ядра для вехи M2; до начала M2 его можно менять свободно.
