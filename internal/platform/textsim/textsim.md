# internal/platform/textsim/textsim.go

Локальный (не-AI, без внешних API) алгоритм сходства текстов для проверки студенческих работ на заимствования: шинглы (n-граммы слов) + MinHash + коэффициент Жаккара. Это платформенный CPU-инструмент — сверка идёт по локальному корпусу прошлых работ; семантическая проверка через embeddings/RAG — отдельный платный контур вне этого пакета.

## Ключевое

- `DefaultShingle = 3` — длина шингла в словах (компромисс между ложными совпадениями и пропуском перефразировок).
- `DefaultSignature = 128` — размер MinHash-сигнатуры (даёт погрешность оценки Жаккара ~±9%).
- `Shingles(text, n) map[uint64]struct{}` — множество хэшей n-грамм слов текста (после нормализации).
- `Jaccard(a, b string) float64` — точный коэффициент Жаккара между двумя текстами: `|A∩B| / |A∪B|`.
- `Signature(text, k) []uint64` — компактная MinHash-сигнатура текста (k минимумов хэш-функций).
- `Estimate(a, b []uint64) float64` — быстрая оценка коэффициента Жаккара по двум сигнатурам (O(k), без обращения к исходным текстам).

## Как это работает

Для каждой сданной работы один раз считается `Signature` и сохраняется в БД; проверка новой работы — сравнение её сигнатуры со всеми ранее сохранёнными через `Estimate` (дёшево, O(k) на пару) для отбора кандидатов, и уже для кандидатов — точный `Jaccard` по полным текстам для финальной проверки. `normalize` приводит текст к сравнимому виду (нижний регистр, только буквы/цифры, «ё»→«е»), чтобы форматирование и пунктуация не влияли на сравнение. Хэш-функции для MinHash — семейство `splitmix64` от `(шингл XOR соль_i)`, детерминированное между запусками и версиями Go.

## Связи

Не имеет внешних зависимостей, кроме стандартной библиотеки (`hash/fnv`, `strings`, `unicode`). Используется модулями учебного процесса (проверка работ студентов) — конкретные вызывающие места не видны из этого файла, но по назначению пакета это модуль вроде `academic`/`tasks`, хранящий подписи работ и запускающий сверку при сдаче новой.

## На что обратить внимание

`Estimate` — вероятностная оценка (может ошибаться на несколько процентов), а `Jaccard` — точный, но дорогой при сравнении со всем корпусом; правильный паттерн использования — сначала `Estimate` по сигнатурам для отсева, потом `Jaccard` по полным текстам только для кандидатов с высокой похожестью.
