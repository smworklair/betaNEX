# internal/platform/httpapi/problem.go

Единый формат ошибок HTTP-API согласно RFC 9457 (Problem Details for HTTP APIs) и вспомогательная функция для успешных JSON-ответов. Это самый часто используемый файл пакета — на него ссылаются практически все остальные обработчики ошибок в проекте.

## Ключевое

- `Problem` — структура тела ошибки: `Type` (URI типа проблемы, по умолчанию `"about:blank"`), `Title` (краткое человекочитаемое описание), `Status` (HTTP-код), `Detail` (подробности).
- `WriteProblem(w http.ResponseWriter, status int, title, detail string)` — пишет ошибку как `application/problem+json`. Основная функция для всех неуспешных ответов во всех модулях.
- `WriteJSON(w http.ResponseWriter, status int, v any)` — пишет успешный ответ как обычный `application/json`.

## Как это работает

Обе функции устанавливают соответствующий `Content-Type`, пишут статус-код и кодируют переданное значение в JSON через `json.NewEncoder`. `WriteProblem` дополнительно оборачивает переданные `title`/`detail` в структуру `Problem` с фиксированным `Type: "about:blank"` — это означает "проблема без специфического типа", допустимое по RFC 9457 значение, когда у ошибки нет отдельно документированной классификации.

## Связи

Зависит только от `encoding/json` и `net/http`. Это базовый строительный блок, на который опирается практически весь пакет `httpapi` (authz.go, auth.go, auditview.go, cors.go, idempotency.go и т.д.) и, судя по всему, HTTP-хендлеры всех доменных модулей (`internal/module/*`) — единый формат ошибок гарантирует, что клиент (SPA на React) всегда разбирает ошибку одинаковым образом независимо от того, какой модуль её вернул.

## На что обратить внимание

Формат `application/problem+json` — это стандарт (RFC 9457), а не изобретение проекта; фронтенд вправе полагаться на то, что у любой ошибки API есть поля `title`/`status`/`detail` в этом виде. Использование единого `Type: "about:blank"` везде означает, что клиент не может различать типы ошибок по этому полю — различение идёт по HTTP-статусу и, при необходимости, по тексту `detail`.
