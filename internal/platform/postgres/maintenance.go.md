# internal/platform/postgres/maintenance.go

Один метод регламентного обслуживания БД — очистка просроченных сессий пользователей. Файл существует, чтобы у платформенного слоя `postgres` было место для фоновых задач обслуживания, отдельное от бизнес-логики модулей.

## Ключевое

- `(*DB) CleanupSessions(ctx) error` — удаляет истёкшие и отозванные сессии из таблицы сессий одним SQL-запросом.

## Как это работает

Метод — тонкая обёртка над сгенерированным sqlc-запросом `db.DeleteExpiredSessions`. Вызывается ночной cron-задачей (планировщиком где-то в `cmd/nexd` или модуле задач), а не по HTTP-запросу пользователя. Количество удалённых строк подавляется (`_ = n`), но по комментарию в коде должно попадать в лог самой cron-задачи, а не логироваться здесь.

## Связи

Зависит от `internal/platform/postgres/db` (сгенерированный sqlc-код с запросом `DeleteExpiredSessions`) и типа `DB` из `postgres.go`. Вызывается из планировщика фоновых задач nexd (не виден напрямую в этом файле).

## На что обратить внимание

Функция не идёт через `InTenantTx`/`InTx` — сессии, судя по всему, общесистемная (не per-tenant в смысле RLS) таблица, поэтому запрос выполняется напрямую на пуле `d.pool`, без установки `app.tenant_id`.
