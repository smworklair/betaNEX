# internal/module/terminal/http.go

Единственный HTTP-вход модуля terminal — командная консоль администратора. Весь функционал терминала выходит наружу через один REST-эндпоинт.

## Ключевое

- `Routes(deps Deps, guard *authz.Guard) func(mux *http.ServeMux)` — монтирует один маршрут: `POST /api/v1/terminal/exec` с телом `{"line": "..."}`.

## Как это работает

Хендлер сначала проверяет право `PermExec` через `guard` — доступ к терминалу целиком закрыт для всех, кроме роли admin. Затем разбирает тело JSON (лимит 64 КБ, `DisallowUnknownFields`) и передаёт строку `line` в `deps.Exec(ctx, line)` — вся логика разбора и исполнения команды находится в `terminal.go`. Ошибки от `Exec` маппятся в HTTP-статусы: `ErrUnknown` → 422 (фронт показывает подсказку набрать `help`), `authz.ErrDenied` → 403 (если конкретная команда внутри требует дополнительное право через шину — например, рассылка уведомлений), любая другая ошибка → 500. Успешный результат — это структура `Result` из `terminal.go`, отдаётся как есть в JSON.

## Связи

Зависит от `internal/kernel/authz` и `internal/platform/httpapi`. Работает как тонкий адаптер над `Deps.Exec` из `terminal.go` того же пакета — вся содержательная логика там.

## На что обратить внимание

Это единственная точка входа для ВСЕХ команд терминала — обзор системы, задачи, финансы, аудит, уведомления и т.д. — в отличие от других модулей, у которых на каждое действие свой REST-маршрут, тут один POST с текстовой строкой команды внутри, разбираемой уже в домене.
