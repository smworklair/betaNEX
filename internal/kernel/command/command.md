# internal/kernel/command/command.go

Определяет базовые интерфейсы шины команд: `Command`, `Handler[C]` и `Bus`. Это контракт (пока черновой, по комментарию — «набросок, не финальный API»), вокруг которого построена вся запись данных в NEX.

## Ключевое

- Package-doc в начале файла — описывает жизненный цикл команды: HTTP-хендлер вызывает `Bus.Dispatch(ctx, cmd)`, дальше идут валидация → авторизация → транзакция → события → аудит.
- `Command` — интерфейс, который должна реализовать любая команда: `Name() string` (стабильное имя для аудита, например `"college.student.enroll"`), `Permission() string` (право, требуемое для исполнения, проверяется `authz`), `Validate() error` (проверка инвариантов входных данных до обращений к БД).
- `Handler[C Command]` — обобщённый интерфейс: `Handle(ctx, cmd C) error`, исполняет команду конкретного типа внутри транзакции, открытой шиной. Регистрируется модулем при старте.
- `Bus` — интерфейс шины: единственный метод `Dispatch(ctx, cmd Command) error`, проводящий команду по полному циклу.

## Как это работает

Файл — это чистые определения интерфейсов без реализации. Реальная логика цикла валидация → авторизация → транзакция → аудит находится в `bus.go` того же пакета (`MemoryBus`). `Command` — минимальный контракт, который должен реализовать каждый тип команды в доменных модулях (`internal/module/*`); он несёт как данные (поля конкретной структуры-команды), так и метаданные о себе (имя, требуемое право, правило валидации).

## Связи

Не зависит ни от чего, кроме `context`. От него зависят `bus.go` (реализация `Bus` через `MemoryBus`), `internal/kernel/authz` (интерфейс `Authorizer` в `bus.go` принимает `command.Command`), и все доменные модули (`internal/module/*`), которые объявляют свои типы команд, реализующие интерфейс `Command`.

## На что обратить внимание

Интерфейс `Handler[C Command]` дженерик, но фактическая шина (`MemoryBus`) работает с нетипизированным `HandlerFunc` — обобщённый `Handler[C]` в этом файле, судя по всему, задаёт желаемую форму API для модулей, а связывание с конкретным именем команды при регистрации всё равно происходит через строковый `Name()` и функцию без дженериков в `bus.go`.
