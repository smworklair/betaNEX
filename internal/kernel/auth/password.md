# internal/kernel/auth/password.go

Хэширование и проверка паролей алгоритмом argon2id с параметрами, соответствующими рекомендациям OWASP Password Storage Cheat Sheet. Результат хранится в стандартном текстовом формате PHC, что позволяет менять параметры стоимости хэширования без миграции уже сохранённых данных.

## Ключевое

- `argonMemoryKiB`, `argonTime`, `argonThreads`, `argonSaltLen`, `argonKeyLen` — параметры argon2id: 19 МиБ памяти, 2 итерации, 1 поток, соль 16 байт, ключ 32 байта.
- `ErrHashFormat` — ошибка, означающая, что хэш в БД не является корректной PHC-строкой argon2id.
- `HashPassword(password string) (string, error)` — генерирует случайную соль, вычисляет argon2id-ключ, возвращает PHC-строку вида `$argon2id$v=19$m=...,t=...,p=...$<salt>$<hash>`.
- `VerifyPassword(password, phc string) (bool, error)` — разбирает PHC-строку, пересчитывает хэш пароля с теми же параметрами/солью и сравнивает результат за постоянное время; ошибку возвращает только при повреждённом/некорректном формате хэша.

## Как это работает

`HashPassword` использует `crypto/rand` для генерации соли и `golang.org/x/crypto/argon2.IDKey` для вычисления хэша, затем кодирует все параметры (версия, память, время, потоки) прямо в саму строку хэша по стандарту PHC — это значит, что при проверке пароля не нужно знать параметры заранее, они читаются из самого хэша. `VerifyPassword` парсит строку через `strings.Split` по `$`, декодирует соль и ожидаемый хэш из base64, пересчитывает `argon2.IDKey` с этими же параметрами и сравнивает байты через `crypto/subtle.ConstantTimeCompare` — сравнение за постоянное время защищает от timing-атак на побайтовое сравнение хэшей.

## Связи

Зависит только от `golang.org/x/crypto/argon2` и стандартной библиотеки. Используется в `auth.go` того же пакета (`Service.Login` вызывает `VerifyPassword`, `dummyHash` вычисляется через `HashPassword`) и в `cmd/nexd/main.go` (CLI-подкоманда `nexd user create` вызывает `auth.HashPassword` при создании пользователя).

## На что обратить внимание

Кодирование параметров хэширования (память/время/потоки/версия) прямо внутри PHC-строки — намеренное проектное решение: если в будущем параметры argon2id ужесточат (например, увеличат память), старые хэши в БД останутся проверяемыми без миграции, так как `VerifyPassword` читает параметры из самого хэша, а не из констант кода.
