# internal/kernel/authz/doc.go

Package-doc файл пакета `authz`, описывающий модель авторизации NEX: RBAC (роль → набор прав) со скоупом на уровне tenant'а, с проверкой на уровне шины команд, а не в HTTP-хендлерах.

## Ключевое

- Комментарий пакета — фиксирует модель авторизации (веха M4): актору назначаются роли, роль — именованный набор прав (permission); каждая команда объявляет требуемое право; проверка выполняется в шине команд до исполнения; решения авторизации (разрешено/отказано и почему) отражаются в аудите; политики — обычный Go-код, покрываемый юнит-тестами (ссылка на ADR-005).

## Как это работает

Файл не содержит кода — только Go-doc над `package authz`, служащий входной точкой понимания для всего пакета (`authorizer.go`, `guard.go`, `policy.go`).

## Связи

Описывает пакет `authz`. Явно указывает на связь с `internal/kernel/command` (проверка авторизации встроена в цикл `Dispatch`) и с `internal/kernel/audit` (решения авторизации логируются).

## На что обратить внимание

Явно подчёркнуто, что проверка прав происходит в шине команд, а не в HTTP-слое — это значит, что любой путь изменения данных (не только HTTP API, но и, например, терминал-консоль или CLI) автоматически проходит одну и ту же авторизацию, если использует `command.Bus.Dispatch`.
