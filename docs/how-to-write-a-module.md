# Как написать модуль NEX

Модуль — самодостаточный срез предметной области (например, `college`: студенты, группы, преподаватели). Этот документ — шаблон; первый настоящий модуль появится на вехе M6 и уточнит его.

## Правила (нерушимые)

Модуль зависит только от `internal/kernel` и `internal/platform` — никогда от другого модуля. Модули общаются между собой событиями, не прямыми вызовами. Любое изменение данных — команда через шину (`command.Bus`); прямых UPDATE из HTTP-хендлеров не бывает. Каждая таблица модуля несёт `tenant_id` и RLS-политику.

## Структура

```
internal/module/college/
  doc.go            — что это за домен, границы ответственности (по-русски)
  commands.go       — типы команд: Name, Permission, Validate
  handlers.go       — HandlerFunc'и: бизнес-логика внутри транзакции
  events.go         — доменные события модуля
  queries/          — не здесь: SQL кладётся в internal/platform/postgres/queries
  http.go           — регистрация маршрутов модуля (тонкий слой: parse → cmd → Dispatch → respond)
  module.go         — Register(...) — единая точка подключения модуля
```

## Пошагово

1. **Спека.** Добавьте маршруты и схемы модуля в `api/openapi.yaml`. Контракт — раньше кода.
2. **Миграция.** Новая миграция в `/migrations`: таблицы с `tenant_id`, RLS-политики, индексы.
3. **SQL.** Запросы в `internal/platform/postgres/queries/<домен>.sql`, затем `make sqlc`.
4. **Команды.** Каждая команда объявляет стабильное имя (`college.student.enroll`), требуемое право (`college:students:write`) и `Validate()` с инвариантами входа.
5. **Хендлеры.** Бизнес-логика; порождают события; не знают про HTTP.
6. **Права.** Объявите права модуля константами; приложение (композиционный корень) раздаёт их ролям через `authz.Policy.Grant`.
7. **HTTP.** Тонкие хендлеры: распарсить вход → собрать команду → `bus.Dispatch` → замапить результат/ошибку (ErrDenied → 403, валидация → 400, RFC 9457).
8. **Регистрация.** `module.Register(bus, mux, ...)` вызывается из `cmd/nexd`.
9. **Тесты.** Юнит — на Validate и хендлеры; интеграционные — через шину с реальным Postgres (testcontainers); негативные тесты изоляции tenant'ов обязательны.

## Имена

Команды: `<модуль>.<сущность>.<глагол>` — `college.student.enroll`. Права: `<модуль>:<сущность>:<действие>` — `college:students:write`. События: `<модуль>.<сущность>.<причастие>` — `college.student.enrolled`.
