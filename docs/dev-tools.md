# NEX — Инструменты разработчика

Что поставить на машину разработчика (Windows) и зачем. Порядок ≈ порядок установки.

## Базовое окружение

- **WSL2 (Ubuntu)** — Go-разработка и Docker на Windows живут лучше в WSL2: нативная файловая система Linux, те же команды, что и на проде. Репозиторий держать внутри WSL (`~/projects`), не на `/mnt/c` — иначе медленный I/O.
- **Windows Terminal** — вкладки PowerShell/WSL, нормальный рендеринг. Стандарт.
- **Git for Windows + git в WSL** — контроль версий. Настроить `git config --global core.autocrlf input` и один `user.email` в обоих окружениях.
- **Docker Desktop** (backend = WSL2) — контейнеры для dev-окружения (`compose.yaml`: Postgres, Valkey) и локальная сборка прод-образов. Альтернатива без лицензионных вопросов — Rancher Desktop.

## Редактор и языки

- **VS Code** + расширения: `golang.go` (gopls, дебаггер delve), `biomejs.biome`, `bradlc.vscode-tailwindcss` (автокомплит классов), `EditorConfig`, `WSL` (удалённая разработка). Альтернатива для Go — **GoLand** (платно, лучший рефакторинг).
- **Go 1.24+** — тулчейн; ставить в WSL через официальный tarball.
- **Node.js LTS + pnpm** — через `corepack enable pnpm`. Версию Node фиксировать в `.nvmrc`/`mise`.
- **mise** (или asdf) — менеджер версий Go/Node/инструментов: одна команда `mise install` даёт всем одинаковые версии.

## Go-инструменты (CLI)

- **golangci-lint** — агрегатор линтеров, гоняется локально и в CI; ловит баги до ревью.
- **gofumpt** — более строгий gofmt; ноль споров о форматировании.
- **air** — hot-reload Go-сервера: сохранил файл — nexd перезапустился.
- **goose** — прогон/откат SQL-миграций локально.
- **sqlc** — генерация Go-кода из SQL после каждого изменения запросов.
- **oapi-codegen** — генерация серверных интерфейсов из `api/openapi.yaml`.
- **govulncheck** — проверка зависимостей на известные уязвимости (и в CI).

## API и база данных

- **Bruno** — клиент API: коллекции хранятся файлами в репозитории (в отличие от облачного Postman) — ревьюятся в PR. Альтернатива: Hoppscotch.
- **psql** — консольный клиент Postgres; быстрее любого GUI для проверки запроса.
- **DBeaver Community** — GUI для схемы, данных, EXPLAIN-планов. Альтернатива: TablePlus (платный, приятнее).
- **pgcli** — psql с автокомплитом и подсветкой.

## Терминальные утилиты

- **jq** — обработка JSON в пайпах (`curl ... | jq .`).
- **httpie** или **curl** — быстрые запросы к API из терминала.
- **lazygit** — TUI для git: staging по строкам, интерактивный rebase без боли.
- **ripgrep (rg)** и **fzf** — мгновенный поиск по коду и fuzzy-поиск файлов.
- **dive** — анализ слоёв Docker-образа (почему образ распух).
- **mkcert** — локальные HTTPS-сертификаты, когда понадобится тестировать secure-cookies.
- **k6** — нагрузочное тестирование API перед релизом.

## Браузер

- **React Developer Tools** — дерево компонентов, профайлер ре-рендеров.
- **TanStack Query/Router Devtools** — встраиваются в приложение в dev-режиме (не расширение) — состояние кэша и роутера.
- Вкладка **Network** + **Lighthouse** в Chrome DevTools — базовый аудит производительности.

## Качество кода (общее)

- **EditorConfig** (`.editorconfig` в корне) — единые отступы/переводы строк для всех редакторов.
- **lefthook** — git-хуки: `gofumpt` + `golangci-lint` + `biome check` на pre-commit; быстрее husky, один бинарник.
- **Biome** — линт+формат TypeScript/JSON одним инструментом (см. ADR-017).
