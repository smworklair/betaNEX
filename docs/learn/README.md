# docs/learn/ — учебный цикл NEX

Шесть документов, чтобы разобраться в проекте с нуля — от «как это
всё вообще устроено» до «как использовать ИИ, чтобы разбираться
быстрее и надёжнее». Это не замена существующей инженерной
документации (`../decision-log.md`, `../go-guide.md`,
`../architecture-go.md`, `../ai/README.md`) — это учебный вход в неё:
меньше «почему» на уровне конкретных решений, больше «с чего начать» и
«как понимать целиком, а не по одной строке».

> **Твой профиль — сильная архитектура, ноль практики?** Если ты
> свободно мыслишь границами модулей, контрактами, потоками данных и
> трейд-оффами, но не знаешь синтаксиса языков, тулинга и того, что для
> джуниора очевидно (пакетные менеджеры, git, как читать стектрейс) —
> начни с [`from-architect-to-code.md`](from-architect-to-code.md).
> Это не курс архитектуры (она у тебя уже есть), а словарь-переводчик
> «концепт, который ты понимаешь → как это записано в коде NEX».
> Дальше читай `how-to-read-and-learn.md` тем же порядком, но держи
> `from-architect-to-code.md` открытым рядом как справочник.

## Рекомендованный порядок чтения

1. **[`getting-started.md`](getting-started.md)** — поднять проект
   руками (Docker или из исходников), переменные окружения, тесты.
   Начните отсюда, если ещё ничего не запускали.
2. **[`how-to-read-and-learn.md`](how-to-read-and-learn.md)** — главный
   документ цикла. Конкретный маршрут чтения кода (день 1 / неделя 1 /
   дальше), что читать до кода, как отличать важное от деталей, как
   осознанно использовать ИИ для разбора незнакомого кода, и
   упражнения от простого до архитектурного.
3. **[`backend-go.md`](backend-go.md)** — гид по бэкенду (Go/nexd):
   слои, поток данных, мультиарендность/RLS, шина команд,
   аутентификация, AI-прокси. С mermaid-диаграммами и картой пакетов.
4. **[`frontend-web.md`](frontend-web.md)** — гид по фронтенду (web/):
   структура после рефакторинга на components/features/pages, роутинг
   на wouter, мини-чат, слой api/, состояние приложения.
5. **[`ai-stack.md`](ai-stack.md)** — гид по AI-части: `ai-gateway`
   (провайдеры, fallback, бюджеты, кэш, стриминг, безопасность) и
   `beta/` (FrankAI/Nex-pilot) как отдельная песочница «понять LLM
   изнутри».
6. **[`from-architect-to-code.md`](from-architect-to-code.md)** —
   мост для тех, у кого архитектурное мышление сильнее практического
   владения синтаксисом/тулингом (см. врезку выше). Читается не вместо
   пунктов 2-5, а параллельно с ними, как словарь.

Документы 3-5 не обязательно читать последовательно — после
`how-to-read-and-learn.md` идите за интересом; порядок здесь просто
логичный (снизу вверх: сначала можно поднять проект, потом понять как
учиться, потом бэкенд → фронтенд → AI).

## Как эти документы связаны с остальной документацией репозитория

```
docs/learn/                    ← учебный вход (вы здесь)
  getting-started.md              практический онбординг
  how-to-read-and-learn.md        маршрут + метод + ИИ как инструмент
  from-architect-to-code.md       мост синтаксис↔архитектура (по профилю)
  backend-go.md          ───┐
  frontend-web.md           │     карта территории для каждой части
  ai-stack.md            ───┘

docs/architecture-go.md        ← более плотный синтез архитектуры Go (без учебных отступлений)
docs/go-guide.md                инженерное руководство: паттерны, решения, roadmap-разделы
docs/decision-log.md            ADR-001…021: конкретные решения и альтернативы
docs/how-to-write-a-module.md   шаблон нового модуля
docs/ai/README.md               канонiчный источник по AI-архитектуре (сегодня/план)
docs/roadmap.md                 вехи M0-M12
backend.md (корень)             план расширения бета-разделов реальными модулями
learning/resources.md           внешние учебные ресурсы (Go, Postgres, React, ...)
```

Если `docs/learn/*` в чём-то разойдётся с кодом — как и везде в этом
репозитории, прав код; поправьте документ PR'ом.

## Для кого это

Для любого, кто открывает этот репозиторий впервые — независимо от
того, начинаете ли вы с Go, с React или с AI-инженерии. Документы
написаны так, чтобы не требовать предварительного знания именно этого
проекта, но раздел «что читать перед кодом» в
`how-to-read-and-learn.md` предполагает базовое знакомство с HTTP,
git и командной строкой. Если этого базового знакомства как раз нет,
а архитектурное мышление есть — это не повод пропускать документы, а
повод начать с `from-architect-to-code.md` (см. врезку выше).
