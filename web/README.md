# web/ — фронтенд NEX

Сейчас здесь лежит **визуальный прототип «КИС Колледж»** (React + Vite), перенесённый из корня репозитория. Он служит референсом дизайна и **не является** целевой архитектурой фронтенда.

По роадмапу (веха M7, см. `../docs/roadmap.md`) прототип будет заменён новым приложением: Vite + TypeScript strict + TanStack Router/Query + Zustand + Tailwind v4 + shadcn/ui, работающим против OpenAPI-контракта из `../api/openapi.yaml`.

## Запуск прототипа

```sh
cd web
npm install   # позже: pnpm install
npm run dev   # http://localhost:3000
```

Прототип обращается к Gemini API — ключ задаётся в `.env` (см. `.env.example`).

## Подключение к бэкенду nexd

Слой доступа к API живёт в `src/api/` — единая точка обращения к серверу
(куки-сессии, разбор ошибок RFC 9457, автофолбэк на моки):

| Файл | Назначение |
| --- | --- |
| `api/client.ts` | Базовый URL, `apiFetch`, `ApiError`, `withFallback` |
| `api/auth.ts`   | `/api/v1/auth/*` — вход, `me`, выход, роли |
| `api/tasks.ts`  | `/api/v1/tasks` — список/создание/выполнение/удаление |
| `api/campus.ts` | `/api/v1/campus/*` — студенты, группы, журнал |
| `api/agents.ts` | пока моки (бэкенд-модуль агентов планируется) |

**Как это работает.** Адрес бэкенда задаётся переменной `VITE_API_URL`
(см. `.env.example`):

- **не задано** → демо-режим: все экраны рисуют встроенные моки, сеть не
  трогается;
- **URL задан** → чтения (`listTasks`, `listStudents`, `listGroups`) идут на
  сервер, а при любой ошибке (нет сети/сессии, 5xx) прозрачно откатываются на
  моки; вход и мутации отражают реальный ответ.

Реально подключены к API экраны **Вход** (сессии `nex_session`), **Задачи** и
**Люди → Студенты/Группы**. Остальные экраны работают на моках, но получают
данные через тот же слой `src/api/` — подключение бэкенда для них не потребует
переписывать компоненты.

### Локально «фронт + бэк» на одном origin

```sh
# терминал 1 — бэкенд (из корня репозитория)
make run                     # nexd на :8080

# терминал 2 — фронт
cd web && npm run dev        # http://localhost:3000
```

Dev-сервер Vite проксирует `/api/*` на nexd (`http://localhost:8080`,
переопределяется `NEX_DEV_PROXY`), поэтому `VITE_API_URL` можно оставить
пустым — CORS не нужен.

### Прод (Vercel)

Go-бэкенд не работает на статическом хостинге Vercel — его деплоят отдельно
(Render/Fly/Railway и т.п.). После этого задайте `VITE_API_URL` в переменных
окружения проекта Vercel (URL живого nexd) и включите на бэкенде CORS с
`credentials` и cookie сессии `SameSite=None; Secure`.
