# web/vite.config.ts

Конфигурация сборщика Vite для фронтенда NEX. Настраивает React и Tailwind CSS плагины, алиас путей, проброс переменной окружения для Gemini API и — что важнее всего для локальной разработки — прокси `/api/*` на бэкенд `nexd`, чтобы фронтенд и бэкенд работали как единый origin без проблем с CORS.

## Ключевое

- `defineConfig(({ mode }) => {...})` — конфиг строится функцией, получающей текущий режим сборки (`development`/`production`), что позволяет подгружать переменные окружения через `loadEnv(mode, '.', '')`.
- `plugins: [react(), tailwindcss()]` — стандартная пара плагинов: официальный плагин React (JSX/Fast Refresh) и Tailwind CSS через Vite-плагин.
- `define: { 'process.env.GEMINI_API_KEY': ... }` — пробрасывает переменную окружения `GEMINI_API_KEY` в клиентский код как строковую константу времени сборки (замена `process.env.GEMINI_API_KEY` на литерал при сборке).
- `resolve.alias['@']` — алиас `@` указывает на корень директории `web/` (`path.resolve(__dirname, '.')`), позволяя импортировать модули по абсолютному пути вида `@/src/...`.
- `server.hmr` / `server.watch` — Hot Module Replacement и файловый вотчер отключаются, если задана переменная окружения `DISABLE_HMR=true` — это сделано специально для среды AI Studio / агентных правок, чтобы избежать «мерцания» интерфейса при частых автоматических изменениях файлов.
- `server.proxy['/api']` — все запросы фронтенда на `/api/*` в dev-режиме перенаправляются на бэкенд `nexd` (по умолчанию `http://localhost:8080`, переопределяется через переменную окружения `NEX_DEV_PROXY`), с `changeOrigin: true`.

## Как это работает

Главная практическая деталь конфига — прокси `/api`: он позволяет фронтенду с пустым `VITE_API_URL` обращаться к реальному бэкенду так, будто это один и тот же сервер (без CORS-заголовков и без указания полного адреса бэкенда в коде фронтенда). Это подтверждено комментарием в самом файле: «Локальная разработка фронт+бэк на одном origin». Второй практический момент — условное отключение HMR/watch: обычно Vite следит за файлами и мгновенно обновляет страницу при изменениях, но когда правки вносит автоматический агент (а не человек в редакторе), частые файловые события создают визуальный шум — поэтому предусмотрен явный флаг для его отключения.

## Связи

Это точка входа сборки для всего `web/`-приложения — фактически определяет, как собираются и на что проксируются все файлы из `web/src/*`, включая все описанные в этом наборе страницы и `terminal.tsx`/`ui.tsx`. Бэкенд, на который проксируются запросы (`nexd`), находится вне `web/` — в серверной части монолита NEX.
