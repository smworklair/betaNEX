# web/src/campus.ts

Модель данных и «ИИ-слой» для раздела «Кампус» (возможности для студентов: стажировки, курсы, гранты, льготы, лекции, достижения, портфолио). Файл специально задуман как прототип будущего backend-контракта: каждый интерфейс — это форма, в которой сервер будет отдавать данные, а каждая функция помечена комментарием `BACKEND:` — местом, где сейчас лежит детерминированный мок, а в будущем будет реальный вызов сервиса или LLM-скоринга.

## Ключевое

- `INTERESTS`, `Interest` — фиксированный список тегов-интересов студента (используется как теги подписки/фильтрации).
- `OppKind`, `OPP_KIND_LABEL`, `Opportunity`, `OPPORTUNITIES` — типы и моковый список «возможностей» (стажировки, вакансии, курсы, хакатоны, гранты, конференции, клубы).
- `Perk`, `PERKS` — студенческие льготы и подписки (например GitHub Student Pack, Яндекс Облако).
- `Lecture`, `LECTURES` — список видео-лекций с прогрессом просмотра.
- `CampusEvent`, `CALENDAR` — события кампуса (экзамены, дедлайны, мероприятия).
- `PROGRESS`, `Achievement`, `ACHIEVEMENTS` — прогресс студента и геймификационные достижения.
- `PortfolioItem`, `PORTFOLIO` — автособираемое портфолио студента.
- `matchScore(opp, interests)` — оценка релевантности возможности интересам студента (0–100), сейчас — простая эвристика по совпадению тегов.
- `whyRecommended(opp, interests)` — человекочитаемое объяснение, почему возможность рекомендована.
- `urgency(days)` — срочность дедлайна (`critical` / `soon` / `ok`).
- `forYou(interests)` — персональная лента «Для вас», отсортированная по релевантности и срочности.
- `continueLecture()` — какую лекцию предложить «продолжить просмотр».

## Как это работает

Все списки данных — статические константы, имитирующие ответы будущих GET-эндпоинтов (`/campus/opportunities`, `/campus/perks` и т.д., см. комментарии над каждым блоком). «ИИ-функции» (`matchScore`, `whyRecommended`, `forYou`) реализуют простую детерминированную логику на JS (пересечение тегов, штраф/бонус за близкий дедлайн) вместо реального ML/LLM-скоринга — но с тем же контрактом вызова, какой будет у реального сервиса, чтобы замена была безболезненной.

## Связи

Не имеет внешних зависимостей внутри проекта (чистый TS-модуль с данными и функциями). Предположительно используется экраном раздела «Кампус» (не входит в текущий список файлов) для отображения ленты возможностей, льгот, лекций и достижений студента.
