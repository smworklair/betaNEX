# web/src/App.tsx

Корень React-приложения NEX: экран логина, глобальная навигация (двухуровневое меню «раздел → подстраницы») и общий каркас (shell) для десктопа и мобильных устройств. Это самый верхний уровень сборки UI — файл решает, что показывать пользователю целиком, а не отдельная бизнес-функциональность.

## Ключевое

- `SECTIONS: Section[]` — дерево навигации: разделы верхней панели (Лента, Финансы, Учёба, Люди, Аналитика, Безопасность, Бета), каждый со своими подстраницами (`SubItem`) и React-узлом страницы.
- `ALL_ITEMS`, `SECTION_OF` — плоские индексы по `id` подстраницы: быстрый доступ к элементу меню и к тому, какому разделу он принадлежит.
- `Login()` — экран входа: два режима — реальный бэкенд (`authApi.authConfigured()` истинно, вход по tenant/email/паролю через `apiLogin`) и демо-режим (выбор роли + пароль `0000`, без сети).
- `NexOmni()` — омнибокс в шапке (Cmd/Ctrl+K): единое поле поиска по разделам, людям и «ИИ-действиям» (`AI_ACTIONS`) плюс прямой вход в чат NEX.
- `Shell()` — десктопный каркас: верхняя панель разделов + боковая панель подстраниц + область контента + горячие клавиши (Ctrl+B — свернуть сайдбар, Ctrl+I — открыть чат, Alt+1..9 — переход по разделам).
- `MobileShell()` — мобильный каркас: верхний бар, чипы подстраниц, нижний докбар (настраиваемый `prefs.dock`), выдвижное меню.
- `App()` (default export) — точка входа: показывает `Login`, если нет `user`, иначе `Shell` или `MobileShell` в зависимости от `useIsMobile()`.

## Как это работает

Навигация не завязана на URL/роутер — единственный источник истины — `page: string` в глобальном состоянии из `useApp()` (контекст в `./ui`), и `renderSub(id)` просто выбирает нужный React-узел по этому `id` из `ALL_ITEMS` (или спецстраницы `chat`/`settings`). Все реальные экраны (`pages/*`) заранее собраны как готовые `<Node/>` внутри `SECTIONS`, то есть при монтировании `App` уже создаются все элементы React-дерева для всех разделов сразу (ленивая загрузка не используется). Поверх каркаса везде висят `ContextDrawer` и `AiLayer` (из `./ai`) — плавающие ИИ-поверхности, не зависящие от текущей страницы.

## Связи

Импортирует контекст приложения и типы из `./ui` (`useApp`, `Beta`, `useIsMobile`, `User`), API-клиент `./api` (`auth`, `ApiError`), конфигурацию навигации из `./dock`, демо-данные и роли из `./data`, ИИ-слой из `./ai`, и все страницы из `./pages/*`. Является корневым компонентом — использует практически весь фронтенд-код проекта транзитивно.

## На что обратить внимание

В демо-режиме кнопки выбора роли, кроме «admin», задизейблены (`frozen`) — это осознанное ограничение прототипа (остальные роли «заморожены»), а не баг верстки. Реальный вход (`apiMode`) и демо-вход — два полностью независимых пути с разными полями формы и разной валидацией в одном компоненте `Login`.
