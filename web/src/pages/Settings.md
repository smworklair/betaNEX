# web/src/pages/Settings.tsx

Экран «Настройки» — центральное место управления профилем, внешним видом, составом навигации и подключением ИИ-провайдера. Один из самых насыщенных «конфигурационных» экранов: почти все переключатели пишут значение в общий объект настроек `prefs` (глобальный контекст приложения, персистится в localStorage) через `setPref`.

## Ключевое

- `Settings` (default export) — весь экран настроек, состоит из последовательности карточек-секций.
- `Row` — локальный переиспользуемый компонент строки настройки: заголовок + описание слева, элементы управления (обычно сегмент-переключатель) справа.
- `ACCENTS` — палитра акцентных цветов темы на выбор пользователя.
- Секция «Профиль» — аватар, имя, роль, кнопка выхода (`setUser(null)`).
- Секция «Оформление» — переключатель светлая/тёмная тема (`theme`/`setTheme`).
- Секция «Персонализация» — цвет акцента, плотность интерфейса, размер шрифта, скругления углов, полоса подсказок NEX, прозрачность — все через `prefs`/`setPref`.
- Секция «Главный экран» — управление блоками главного экрана (`HOME_BLOCK_CATALOG`/`homeBlocks`), стилем ярлыков, часами, подсказками, обращением в приветствии; кнопка «Настроить экран» переключает `setHomeEditing(true)` и переходит на страницу `home` — то есть открывает конструктор прямо на месте.
- Секция «Рабочая область» — свернуть/развернуть боковую панель, справка по горячим клавишам.
- Секция «Верхняя панель» / «Нижняя панель» — включение/выключение пунктов навигации (`TOPBAR_CATALOG`/`DOCK_CATALOG` из `dock.ts`) с ограничениями `TOPBAR_MIN`/`DOCK_MIN`/`DOCK_MAX`.
- Секция «Интеллект» — статус подключения к `ai-gateway` (`checkGateway()`) и выбор провайдера из списка, реально настроенного на сервере (`fetchProviders()`, `GET /api/v1/ai/providers`). Ключей здесь больше нет — они живут только в переменных окружения `ai-gateway`.
- Секция «Агенты NEX» — ссылка на страницу `agents` (помечена как `Beta`).
- Секция «Навигация и ИИ» — включение ИИ-режима без боковой панели, «пульс агентов» в шапке, кнопка открытия полного чата (`openChat()`).
- Секция «Безопасность» — 2FA и смена пароля (помечены `Soon` — заглушки, не реализовано).

## Как это работает

Большинство настроек читаются и пишутся через `prefs`/`setPref` из `useApp()` — единая точка хранения предпочтений пользователя. Настройки навигации (`dock`, `topbar`, `homeBlocks`) хранятся как массивы id с защитой минимума/максимума элементов (`toggleDock`, `toggleTop`, `toggleHomeBlock` проверяют границы перед изменением). Блок «Интеллект» при монтировании (если `llmReady()`) параллельно проверяет доступность шлюза (`checkGateway()`) и запрашивает список провайдеров (`fetchProviders()`), затем просто хранит выбор имени провайдера (`setProvider`) — никакой проверки/сохранения секретов здесь больше нет.

## Связи

Зависит от `web/src/ui.tsx` (`useApp`, `PageHead`, `Chip`, `Avatar`, `Soon`, `Beta`, тип `Prefs`), `web/src/dock.ts` (каталоги верхней/нижней навигации), `web/src/home.ts` (каталог блоков главного экрана), `web/src/data.ts` (`roleLabel`), `web/src/llm.ts` (выбор провайдера, статус и список провайдеров `ai-gateway`). Является управляющим центром для многих других экранов: то, что здесь настроено, влияет на `web/src/pages/Home.tsx` (состав блоков), навигационные компоненты (верхняя панель/докбар) и на то, какой движок отвечает в `web/src/pages/Chat.tsx`.
