# web/src/pages/Chat.tsx

Экран полноценного чата с ИИ-помощником NEX (страница «Чат» в навигации). Отличается от встроенного мини-терминала (`web/src/terminal.tsx`) тем, что это полноэкранный диалог в стиле ChatGPT/Gemini: пустое состояние с приветствием и подсказками, лента сообщений, действия над сообщениями (копировать, лайк/дизлайк, ответвить чат), агентные «выполнения задач» с видимыми шагами и упоминания разделов через `@`.

## Ключевое

- `Chat` (default export) — сам экран: управляет вводом, историей сообщений (`chatLog` из глобального состояния `useApp`), пустым/заполненным состоянием.
- `send(q)` — основная функция отправки вопроса: если в `llm.ts` настроен и готов реальный LLM-провайдер (`llmReady()`), уходит к нему с историей последних 8 сообщений; при ошибке или отсутствии ключа откатывается на локальный мок-движок `nexReply` из `nexbrain.ts`.
- `AgentRun({ label })` — визуализация «агентного выполнения задачи»: показывает список шагов из `planFor(label)` (`nexbrain.ts`) и постепенно, по таймеру, отмечает их выполненными — имитация того, что NEX «делает» что-то по шагам, а не просто отвечает текстом.
- `NavChips({ nav })` — кнопки-переходы под ответом NEX, ведущие в конкретные разделы приложения через `setPage`.
- `SUGGEST` — список стартовых подсказок-вопросов на пустом экране.
- `MENTIONS` — список разделов, которые можно подключить к контексту вопроса через `@`.

## Как это работает

История чата (`chatLog`) хранится в общем контексте приложения (`useApp()`), поэтому переживает переход между страницами. При отправке сообщения сразу добавляется реплика пользователя, затем — либо запрос к настоящему LLM с плейсхолдером `pending: true` (анимация «печатает»), либо мгновенный ответ мок-движка. Ответ NEX может содержать `data` (структурированный блок — рендерится через `DataBlock` из `nexdata.tsx`), `nav` (кнопки перехода) и `action` (кнопка, запускающая `AgentRun`). Текст ответа рендерится через собственный markdown-компонент `Md` (`web/src/md.tsx`). `pendingAsk`/`clearPendingAsk` из `useApp()` позволяют другим частям приложения (например кнопке «Спросить NEX» на карточке студента) программно отправить готовый вопрос в чат при переходе на эту страницу.

## Связи

Зависит от `web/src/ui.tsx` (`useApp`, `Chip`), `web/src/nexbrain.ts` (`nexReply`, `planFor`, тип `NavLink`), `web/src/llm.ts` (`llmReady`, `llmAsk`, тип `LlmTurn`), `web/src/md.tsx` (`Md`), `web/src/nexdata.tsx` (`DataBlock`). Является одним из двух мест использования единого «мозга» NEX — второе место, судя по комментариям в `nexbrain.ts`, встроенный терминал на главном экране (`web/src/pages/Home.tsx` / `web/src/terminal.tsx`).
