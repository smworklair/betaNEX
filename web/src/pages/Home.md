# web/src/pages/Home.tsx

Главный экран приложения («Главное» в навигации) — то, что видит пользователь сразу после входа. Для роли `admin` это персонализируемое «рабочее место» с приветствием, часами и встроенным командным терминалом NEX (`CommandDeck`), которое можно перестраивать под себя (конструктор блоков и ярлыков). Для всех остальных ролей (преподаватель, бухгалтер, студент) — облегчённая, некастомизируемая «спокойная сводка дня» (`CalmHome`) с карточками дел и полем вопроса к NEX. Это один из крупнейших и самых сложных файлов фронтенда: в нём совмещены конструктор дашборда, встроенный командный терминал с историей и подсказками, интеграция с локальным и настоящим LLM.

## Ключевое

- `Home` (default export) — точка входа: `CommandDeck` для админа, `CalmHome` для остальных ролей.
- `CommandDeck` — редактируемый главный экран администратора: приветствие, панель конструктора (`homeEditing`), рендер и переупорядочивание блоков (`brief`, `console`, `shortcuts`, `today`) и ярлыков по настройкам из `prefs` (`web/src/ui.tsx`) и каталогов из `web/src/home.ts`.
- `NexTerminal` — встроенный компактный терминал NEX: поле ввода команды, журнал сессии, история по стрелкам вверх/вниз, автодополнение команд через `/`-меню, три уровня обработки ввода (см. «Как это работает»).
- `CalmHome` — некастомизируемая сводка дня для не-админских ролей: список карточек-дел с переходом в разделы и простое поле вопроса к NEX (открывает полный чат `openChat`).
- `LiveClock` — тихие часы в шапке `CommandDeck`, обновляются раз в 30 секунд (только минуты, без секунд).
- `greeting()` — подбирает приветствие и иконку по текущему часу суток (ночь/утро/день/вечер).
- `TermMsg`, `TERM_KEY`, `TERM_LIMIT` — тип сообщения терминала и константы для хранения журнала сессии в `sessionStorage` (переживает переход между страницами, но не перезагрузку вкладки; лимит 60 строк).
- `STATUS_COMMANDS` — словарь именованных команд состояния (`status`/`risk`/`finance`/`security` и их русские синонимы), которые отвечают мгновенно без ИИ, готовым срезом данных.
- `ECO_TOKENS` (`TERMINAL_BACKEND_TOKENS`) — токены команд «экосистемы», которые при настроенном backend (`API_CONFIGURED`) уходят на сервер через `POST /terminal/exec`, а не обрабатываются локальным мок-движком.
- `TERM_HELP` — текст справки терминала (`help`).
- `loadTermSession()` — восстанавливает журнал и историю команд терминала из `sessionStorage` при монтировании, отбрасывая «зависшие» pending-сообщения от прерванных запросов.
- `TermEcho` — подсвечивает распознанное имя команды в эхо введённой строки.
- `openSection(name)` (внутри `NexTerminal`) — реализация команды `open <раздел>`: ищет совпадение по названиям ярлыков и заголовков страниц, переключает `setPage` с небольшой задержкой (чтобы журнал успел сохраниться перед размонтированием).
- `run(raw)` (внутри `NexTerminal`) — центральная функция исполнения команды терминала.

## Как это работает

`CommandDeck` держит состав и порядок блоков/ярлыков в `prefs.homeBlocks`/`prefs.homeShortcuts` (глобальные настройки пользователя, см. `web/src/pages/Settings.tsx` и `web/src/ui.tsx`) и переключается в режим редактирования (`homeEditing`, тоже часть глобального состояния приложения) — в этом режиме у каждого блока появляется панель «выше/ниже/скрыть», а у ярлыков — «левее/правее/убрать», плюс возможность вернуть скрытые блоки/ярлыки. Выход с главного экрана автоматически завершает режим редактирования (`useEffect` с очисткой при размонтировании).

Терминал `NexTerminal` — это фактически мини-REPL с несколькими уровнями обработки ввода команды `run(cmd)`, проверяемыми по порядку:
1. Служебные команды (`clear`/`очистить`, `help`/`?`/`помощь`, `open <раздел>`/`открой <раздел>`) — обрабатываются напрямую в этом файле.
2. Если backend настроен (`API_CONFIGURED`) и введённая команда — один из «экосистемных» токенов (`ECO_TOKENS`), запрос уходит на сервер через `terminalExec` (`web/src/api/terminal.ts`) — это реальные данные с аудитом на стороне backend.
3. Именованные команды состояния (`STATUS_COMMANDS`) — отвечают мгновенно локальным мок-движком `nexReply` (`web/src/features/ai/nexbrain.ts`) без обращения к LLM, возвращая готовый срез данных (таблица риска, финансовая сводка, KPI, статус безопасности через `DataBlock`).
4. Общий реестр команд `execRegistry`/`TERM_COMMANDS` из `web/src/features/terminal/terminal.tsx` (например работа с задачами через `useCollection<TermTask>('tasks', ...)` из `web/src/beta/store.ts` — задачи здесь всё ещё локальный бета-слой).
5. Если ничего не распознано как команда — это свободный вопрос к NEX: если в `web/src/features/ai/llm.ts` настроен реальный провайдер (`llmReady()`), запрос уходит туда с системным промптом «ты терминал»; при ошибке или отсутствии ключа — откат на локальный `nexReply`.

Терминал хранит журнал и историю команд в `sessionStorage` под ключом `nex-terminal-log`, поэтому диалог переживает переходы между разделами приложения в пределах вкладки, но очищается при закрытии/перезагрузке. Есть также «полноэкранная среда» терминала (`TerminalWorkspace`, тоже из `web/src/features/terminal/terminal.tsx`), открывающаяся по кнопке — использует тот же `engineCtx` и, если backend настроен, тот же `terminalExec` как удалённый исполнитель.

`CalmHome` намного проще: три статичные карточки дел (финансы, риск студентов, задачи), собранные из моковых данных (`finance`, `aiInsights` из `data.ts`), и форма, которая просто передаёт введённый текст в `openChat(q)` — то есть открывает полноценный чат (`web/src/pages/Chat.tsx`) с этим вопросом.

## Связи

Самый насыщенный по числу зависимостей файл среди рассмотренных: `web/src/ui.tsx` (`useApp` — почти всё глобальное состояние: `prefs`, `homeEditing`, `openChat`, `setPage`, `toast`), `web/src/data.ts` (`finance`, `aiInsights`, `failedLogins`, `students`), `web/src/features/ai/nexbrain.ts` (`attendanceRate`, `nexReply`, `PAGE_TITLES`, типы `NavLink`/`NexData`), `web/src/features/ai/llm.ts` (`llmReady`, `llmAsk`), `web/src/components/md.tsx` (`Md`), `web/src/features/ai/nexdata.tsx` (`DataBlock`), `web/src/beta/store.ts` (`useCollection`), `web/src/pages/tasks` (`TASK_SEED` — не входит в текущий список файлов), `web/src/api/terminal.ts` (`terminalExec`, `TERMINAL_BACKEND_TOKENS`), `web/src/api/client.ts` (`API_CONFIGURED`, `ApiError`), `web/src/features/terminal/terminal.tsx` (весь общий движок команд: `TerminalWorkspace`, `TermResBlock`, `execRegistry`, `TERM_COMMANDS`, `searchCommands`, `CmdMenu` и связанные типы), `web/src/home.ts` (каталоги блоков/ярлыков и `moveInArray`). Настройки, отображаемые здесь (состав блоков, ярлыков, имя приветствия), редактируются также из `web/src/pages/Settings.tsx` через общий `prefs`.

## На что обратить внимание

Терминал на «Главном» и полноэкранная `TerminalWorkspace` из `web/src/features/terminal/terminal.tsx` используют один и тот же `execRegistry`/`engineCtx` — это намеренно единый движок команд, а не два независимых. Также обратите внимание на порядок проверок в `run()`: backend-команды (`ECO_TOKENS`) перехватываются раньше локальных мок-команд состояния именно для того, чтобы при настоящем backend слова вроде «финансы»/«безопасность» показывали реальные серверные данные, а не демо-срез.
