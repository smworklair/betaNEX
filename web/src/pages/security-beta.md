# web/src/pages/security-beta.tsx

Бета-версия раздела «Безопасность»: учётные записи, роли и права, журнал аудита, сессии и устройства, API-ключи/OAuth/SSO, политики безопасности (MFA, IP-ограничения), резервные копии и сертификаты. Самый широкий по охвату бета-раздел из всех — семь независимых экранов в одном файле, большинство на связке `useCollection` + `EntityManager`.

## Ключевое

- `SecUsers()` — учётные записи (имя, email, роль, MFA, статус) через `EntityManager`.
- `SecRoles()` — матрица прав: таблица «право × роль» с чекбоксами `toggle`, добавление/удаление ролей — единственный экран файла с ручной (не через `EntityManager`) табличной вёрсткой, потому что нужна матрица, а не список строк.
- `SecAudit()` — журнал аудита событий (`auditEvents` из `../data` как сид).
- `SecSessions()` — активные сессии (с завершением сессии кнопкой) + `EntityManager` для учёта устройств.
- `SecKeys()` — API-ключи (создание/копирование/отзыв через `KeyCreate`-модалку), переключатели OAuth-провайдеров (Google, Яндекс) и SSO (SAML).
- `SecPolicies()` — тумблеры политик (обязательная MFA, ограничение по IP, сложность паролей, автовыход) + `EntityManager` для белых/чёрных списков IP.
- `SecBackup()` — резервные копии (создание/восстановление/удаление), `EntityManager` для сертификатов, плитки мониторинга целостности сервисов (`services` из `../data`).
- `Screen({ title, sub, icon, children, ask })` и `ToggleRow({ label, desc, on, onToggle })` — общие внутренние хелперы: обёртка экрана с вводной карточкой NEX и переиспользуемая строка-тумблер.

## Как это работает

Общий паттерн такой же, как в `finance-beta.tsx`/`people-beta.tsx`: сид-массив → `useCollection('sec-...', SEED)` → `EntityManager` с описанием колонок/полей. Отличие в том, что здесь есть экраны с нестандартным UI, не сводимым к простой таблице: матрица прав в `SecRoles` (чекбоксы на пересечении роли и разрешения) и наборы тумблеров в `SecKeys`/`SecPolicies` (обычный `useState`, не персистентный между сессиями, кроме сущностей через `useCollection`). Вводная карточка `Screen` на каждом экране прямо говорит: ИИ здесь помогает замечать аномалии, но решает всё политика/роли/журналы, а реальное применение (блокировки, выпуск ключей, бэкапы) — работа бэкенда.

## Связи

Зависит от `../ui` (`PageHead`, `NexAsk`, `Beta`, `Chip`, `Avatar`, `useApp`), `../beta/store` (`useCollection`, `uid`, `humanTime`, `Entity`), `../beta/manager` (`EntityManager`, `Col`, `FieldDef`), `../beta/kit` (`Modal`, `Field`, `Text`, `Select`, `AddButton`, `Confirm`) и `../data` (`sessions as seedSessions`, `auditEvents`, `services`, `roleLabel`). У обычного (не бета) раздела безопасности отдельного файла в этом списке нет — `security-beta.tsx` пока единственная реализация раздела.
