# web/src/pages/academic-beta.tsx

Набор бета-экранов учебного администрирования, которых ещё нет на реальном backend: дисциплины, домашние задания, экзаменационные/зачётные ведомости, учебные планы, приказы по студенческому составу. В отличие от `web/src/pages/academic.tsx` (расписание, журнал, посещаемость — уже используют моковые, но «настоящие» по структуре данные `data.ts`), этот файл целиком построен на бета-инфраструктуре: локальное хранилище `beta/store.ts` и универсальный CRUD-конструктор `beta/manager.tsx`.

## Ключевое

- `Discipline`, `Homework`, `Sheet`, `Curriculum`, `Order` — типы сущностей раздела (каждая расширяет `Entity` из `beta/store.ts`).
- `SEED_DISC`, `SEED_HW`, `SEED_SHEET`, `SEED_CUR`, `SEED_ORD` — начальные (seed) данные для соответствующих коллекций localStorage.
- `Screen` — локальная обёртка экрана: заголовок с иконкой, пометка `Beta`, карточка-подсказка NEX с кнопкой «Разобрать» (`NexAsk`).
- `cols`/`flds` — крошечные типизированные хелперы-фабрики массивов (`Col<T>[]`, `FieldDef<T>[]`) для более компактного описания колонок/полей формы при вызове `EntityManager`.
- `Disciplines`, `Homeworks`, `GradeSheets`, `Curricula`, `Orders` — пять экспортируемых экранов-страниц, каждый — тонкая обвязка вокруг `EntityManager` со своей коллекцией (`useCollection`), колонками и полями формы.
- `ACAD_ICON` — переэкспорт иконки `ClipboardList`, чтобы не дублировать импорт в `App.tsx` при регистрации маршрутов этого раздела.

## Как это работает

Каждая из пяти страниц — это несколько строк: получить коллекцию через `useCollection<T>('acad-...', SEED)` и передать её в `EntityManager` вместе с описанием колонок таблицы и полей формы редактирования. Вся логика CRUD, поиска, экспорта/импорта, массового удаления уже реализована в `EntityManager` — здесь только декларация «что показывать». Данные живут в localStorage браузера (ключи `nex-beta:acad-disciplines` и т.п.), значит не синхронизируются между пользователями/устройствами — это временное решение до появления backend-модуля (см. упоминание `backend.md` в подсказке NEX).

## Связи

Зависит от `web/src/ui.tsx` (`PageHead`, `NexAsk`, `Beta`), `web/src/beta/store.ts` (`useCollection`, `Entity`), `web/src/beta/manager.tsx` (`EntityManager`, `Col`, `FieldDef`), `web/src/data.ts` (`groups` — список групп для выпадающих списков). Пять экспортируемых функций подключаются в `App.tsx` как отдельные маршруты бета-версии учебного раздела, параллельно с `web/src/pages/academic.tsx`.
