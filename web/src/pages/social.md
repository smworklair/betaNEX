# web/src/pages/social.tsx

Самый большой «коммуникационный» раздел приложения: лента-сообщество (внутренняя соцсеть колледжа), полноценный мессенджер с почтовым клиентом, плавающий мини-мессенджер (боковой вылет поверх любой страницы) и рассылка официальных сообщений группам. Файл объединяет несколько разных UX-поверхностей одного и того же коммуникационного слоя данных из `../data`.

## Ключевое

- `Community()` — лента-соцсеть: композер поста, фильтры по типу (Официально/Преподаватели/Сообщество), карточки постов с лайками и раскрывающимися комментариями.
- `Mail()` — главный экспорт для полноэкранного экрана «Сообщения»: переключает десктопный/мобильный вид, десктоп умеет разворачиваться в истинный полноэкранный режим (`fs`, портал в `document.body`, выход по Esc).
- `DesktopMessenger` / `MobileMessenger` — режим «Чат»: список чатов + окно переписки; на мобильном список и диалог показываются раздельно с кнопкой «назад».
- `MailClient` — режим «Почта»: полноценный почтовый клиент (папки, поиск, избранное/звёздочки, чтение письма, ответ/пересылка, `MailCompose` — окно написания письма с кнопкой «Составить с NEX»).
- `MiniMessenger()` — плавающая кнопка (`mini-fab`) с бейджем непрочитанных, разворачивающаяся в компактную панель чата/почты без ухода со страницы; кнопка «во весь экран» переводит на `Mail()` через `setPage('mail')`.
- `Broadcast()` — рассылка официальных объявлений по группам/преподавателям, отдельная от личных чатов, с кнопкой «Составить с NEX» и историей рассылок.
- `ConvHeader` / `ConvBody` / `ChatList` / `ModeToggle` — общие внутренние строительные блоки, переиспользуемые и в `DesktopMessenger`, и в `MiniMessenger`.

## Как это работает

`Mail()` и `MiniMessenger()` — две независимые точки входа в один и тот же набор данных чатов/почты (`chats`, `threads`, `thread`, `emails` из `../data`), поэтому логика диалога (`ConvBody`) написана один раз и используется в обоих местах. Полноэкранный режим (`fs` в `Mail`) не меняет DOM-иерархию компонента, а оборачивает уже готовый JSX в `createPortal(..., document.body)` — так `position: fixed` не ломается о `backdrop-filter` родительских карточек (это явно прокомментировано в коде). Все «отправки» (сообщений, писем, рассылок, комментариев, лайков) в реальности не отправляются никуда — только показывают `toast` через `useApp()`, потому что раздел работает без бэкенда.

## Связи

Зависит от `../ui` (`PageHead`, `Avatar`, `useApp`, `useIsMobile`) и `../data` (`posts`, `postComments`, `chats`, `thread`, `threads`, `groups`, `emails`, `MAIL_FOLDERS`, типы `PostKind`/`MailFolder`/`Email`). `MiniMessenger` использует `setPage`/`page` из `useApp()`, чтобы переключать на полноэкранный маршрут `mail` — то есть связан с общей системой навигации приложения, а не только локальным состоянием.
