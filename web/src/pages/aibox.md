# web/src/pages/aibox.tsx

Небольшой переиспользуемый компонент `AiBox` — встраиваемый мини-чат «спроси ИИ» для раздела приложения. Вместо того чтобы на каждой странице (приём, финансы, календарь, облако и т.д.) писать свою форму общения с ИИ и свой системный промпт строкой, разработчики ставят `<AiBox page="..." facts={[...]} />` — роль ассистента для раздела достаётся с сервера (`ai-gateway/app/core/context_registry.py`), а компонент сам решает, обращаться ли к живой модели через `ai-gateway` или показывать демо-ответ.

## Ключевое

- `AiBox({ title, placeholder, quick, page, facts, system, fallback })` — единственный экспорт файла. Пропсы:
  - `title` — заголовок карточки (и человекочитаемое имя контекста, уходит в `PageContext.title`);
  - `placeholder` — текст-подсказка в поле ввода;
  - `quick` — необязательный массив быстрых вопросов (кнопки-чипы над полем ввода);
  - `page` — идентификатор раздела для реестра контекста на сервере (тот же, что в `nexbrain.ts:PAGE_TITLES`, например `'finance'`);
  - `facts` — короткие ЖИВЫЕ данные экрана (не роль, а факты — например текущая задолженность);
  - `system` — явный оверрайд системного промпта, для случаев вне реестра разделов (используется редко);
  - `fallback` — функция `(q: string) => string`, генерирующая правдоподобный демо-ответ, если ИИ не подключён.

## Как это работает

Компонент хранит короткий тред сообщений (`turns`) в `useState`, инициализированный из `sessionStorage` по ключу `nex-ai-hist:<page>` — так история своя на каждый раздел и переживает размонтирование/повторное открытие компонента в рамках вкладки браузера (но не переживает закрытие вкладки — это осознанно лёгкая память, а не полноценная БД-история). При отправке вопроса (`ask`) сначала проверяется `llmReady()` (сконфигурирован ли `ai-gateway`). Если да — вызывается `llmAsk(q, { history, system, context: { page, title, facts } })` и ответ добавляется в тред через рендерер `Md`. Если ИИ не подключён или запрос упал с ошибкой — тихий откат на `fallback(q)`. Пока идёт запрос — индикатор «печатает» (`typing`).

## Связи

Зависит от `../features/ai/llm` (`llmReady`, `llmAsk`, `LlmTurn` — обёртка над вызовом `ai-gateway`) и `../components/md` (`Md` — рендер markdown-ответов). Используется как строительный блок на нескольких страницах — например, `pages/beta.tsx` (Calendar, Cloud) и `pages/operations.tsx` (Admissions, Finance) — везде, где нужен лёгкий мини-чат с памятью на свой раздел, но без полноценной истории уровня главного чата NEX (`Chat.tsx`).

## На что обратить внимание

История `AiBox` — per-`page`, а не глобальная: два экземпляра с одинаковым `page` на разных страницах будут делить одну ветку (в проекте такого не происходит — у каждого вызова свой уникальный `page`). Если на одной странице открыто несколько `AiBox`, им нужны разные `page`, иначе истории перемешаются.
