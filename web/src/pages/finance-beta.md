# web/src/pages/finance-beta.tsx

Бета-версия бухгалтерского учёта: набор классических «документных» экранов (счета, счета-фактуры, акты, договоры, дебиторка/кредиторка, касса, банк, журнал проводок), каждый — CRUD-таблица поверх универсального `EntityManager`. Это низкоуровневый учётный слой, в отличие от `finance-pro.tsx`, где акцент на аналитике и ИИ-копайлоте.

## Ключевое

- `FinInvoices`, `FinVat`, `FinActs`, `FinContracts`, `FinReceivables`, `FinPayables`, `FinCashbook`, `FinBank`, `FinJournal` — девять экспортируемых компонентов-экранов, каждый — тонкая обёртка над `EntityManager` со своим набором колонок/полей и сид-данными.
- `Screen({ title, sub, icon, children, ask })` — общая обёртка экрана: заголовок страницы + вводная карточка NEX, поясняющая, что раздел работает на локальных бета-данных, а реальные проводки/обмен с банком и ФНС — задача бэкенда.
- Типы документов (`Invoice`, `Vat`, `Act`, `Contract`, `Cash`, `Bank`, `Op`, `Debt`) — каждый расширяет `Entity` и `Record<string, unknown>` (нужно для совместимости с generic `EntityManager`).
- `cols<T>()` / `flds<T>()` — маленькие type-safe хелперы для типизированного построения массивов `columns`/`fields`, передаваемых в `EntityManager`.

## Как это работает

Каждый экран — это связка «сид-данные → `useCollection('fin-...', SEED)` → `EntityManager` с описанием колонок и полей формы». Вся логика создания/редактирования/удаления/поиска/экспорта CSV/импорта JSON инкапсулирована внутри `EntityManager` (модуль `../beta/manager`) — сами компоненты этого файла только объявляют, какие поля показывать и как их редактировать (`type: 'number' | 'date'`, `options: [...]` для select). `FinReceivables`/`FinPayables`/`FinCashbook` дополнительно считают агрегаты (сумму задолженности, приход/расход) прямо в рендере через `.reduce()` по данным коллекции.

## Связи

Зависит от `../ui` (`PageHead`, `NexAsk`, `Beta`), бета-инфраструктуры `../beta/store` (`useCollection`, `Entity`) и `../beta/manager` (`EntityManager`, `rub` — форматирование рублей, типы `Col`/`FieldDef`). Это низкоуровневый «документный» аналог финансового раздела рядом с `finance-pro.tsx`, где живут дашборд, сверка с банком, закрытие периода и сценарное моделирование.
