# web/src/pages/tasks.tsx

Полнофункциональный бета-планировщик задач: подзадачи, чек-листы, исполнители/наблюдатели, повторы, теги, вложения, комментарии, история изменений, массовые операции, шаблоны задач и правила автосоздания задач по событиям в системе. Самый богатый по модели данных бета-раздел приложения.

## Ключевое

- `Tasks()` — корневой экспорт с тремя вкладками: «Список», «Шаблоны», «Правила».
- `BTask` (интерфейс, расширяет `Entity`) — полная модель задачи: статус, приоритет, категория, теги, дедлайн, исполнители, наблюдатели, повтор, подзадачи, чек-лист, комментарии, история, вложения.
- `TaskList` — таблица-список с поиском, фильтрами (статус/приоритет/категория), множественным выбором (`BulkBar` — массовое завершение/повышение приоритета/рассылка/удаление) и деталями задачи по клику.
- `TaskEditor` — модальная форма редактирования всех полей задачи, включая вложенные редакторы `ChecklistEditor` и `SubtaskEditor`.
- `TaskDetail` — карточка-разворот задачи: чипы приоритета/срока/повтора, исполнители/наблюдатели, подзадачи, чек-лист, вложения, комментарии (с добавлением от текущего пользователя) и таймлайн истории изменений.
- `SendDialog` — диалог рассылки задачи одному человеку / группе / нескольким получателям.
- `Templates` / `TemplateEditor` — шаблоны задач (`Template`, коллекция `'task-templates'`), из которых одной кнопкой создаётся новая задача с предзаполненным чек-листом.
- `Rules` / `RuleEditor` — правила автосоздания задач по триггерам (`Rule`, коллекция `'task-rules'`) — например «Платёж просрочен более 3 дней → задача бухгалтеру»; пока это только настройка условий, исполнение остаётся за бэкендом.
- `TASK_SEED` — экспортируемый сид задач, преобразованный из общего мокового массива `seedTasks` (`../data`) в богатую модель `BTask`; специально экспортирован, так как терминал (`terminal.tsx`) читает и пишет в ту же коллекцию `'tasks'`, что и этот раздел — «одна экосистема, а не отдельные копии данных» (комментарий в коде).

## Как это работает

Все данные (задачи, шаблоны, правила) живут в трёх независимых коллекциях `useCollection` (`'tasks'`, `'task-templates'`, `'task-rules'`). Каждое изменение задачи (создание, редактирование, комментарий, переключение чек-листа/подзадачи) дописывает запись в поле `history` — так карточка задачи хранит собственный аудит-лог. Массовые операции (`BulkBar`) работают через `col.patchMany(ids, patch)` — один вызов обновляет сразу несколько записей коллекции. Использование шаблона (`useTemplate`) создаёт новую задачу, копируя заголовок/приоритет/категорию и превращая строки чек-листа шаблона в полноценные объекты `ChecklistItem` с уникальными id.

## Связи

Зависит от `../ui` (`PageHead`, `Chip`, `NexAsk`, `Beta`, `useApp`), `../data` (`tasks as seedTasks`, `staff`, `groups`), бета-инфраструктуры `../beta/store` (`useCollection`, `uid`, `nowIso`, `humanTime`, `Entity`) и обширного набора форм-примитивов `../beta/kit` (`Modal`, `Field`, `Text`, `Area`, `Select`, `TagInput`, `MultiSelect`, `Toolbar`, `AddButton`, `BulkBar`, `Confirm`, `Empty`, `RowCheck`). Коллекцию `'tasks'` совместно использует `web/src/terminal.tsx` — команды терминала `задачи`/`новая задача`/`готово` читают и пишут в те же самые записи, что видны на этой странице.
