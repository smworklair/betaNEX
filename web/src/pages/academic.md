# web/src/pages/academic.tsx

Три «настоящих» (немного более зрелых, чем бета) экрана учебного процесса: расписание занятий, журнал оценок с интерактивным выставлением оценок и посещаемость по группам. В отличие от `academic-beta.tsx`, здесь нет CRUD через `beta/manager` — экраны читают структурированные моковые данные напрямую из `data.ts` и держат небольшое собственное состояние (выбранная группа/предмет, локальные правки оценок).

## Ключевое

- `GroupSelect` — общий для всех трёх экранов выпадающий список групп.
- `Schedule` — расписание занятий выбранной группы на неделю: таблица дни×время из `scheduleSlots`/`scheduleDays`, плюс карточка-подсказка NEX о проверке конфликтов и свободных окнах.
- `Journal` — журнал оценок: таблица студент×дата занятия, оценки кликабельны и циклически меняются по клику (`cycle`: 5→4→3→2→пропуск→5, локальный стейт `edits` поверх сгенерированных `gradesFor`), подсчёт среднего балла на лету, кнопка «Уведомить» для студентов со средним ниже 3.5.
- `Attendance` — таблица посещаемости группы с прогресс-баром (`meter`) и цветовой индикацией риска (норма/внимание/риск), кнопка «Уведомить» через NEX для студентов с посещаемостью ниже 85%.
- `LESSON_DATES` — захардкоженный список дат последних 6 занятий (для заголовков колонок журнала).

## Как это работает

Ни один из трёх экранов не сохраняет изменения — «Сохранить» в `Journal` и «Уведомить» везде вызывают `toast(...)` без реальной мутации данных (`edits` в журнале живёт только в React-состоянии компонента и теряется при смене группы или перезагрузке). Все данные (студенты, группы, предметы, оценки, расписание) — статические моковые константы/генераторы из `web/src/data.ts`; `gradesFor(group)` — детерминированная псевдослучайная функция, дающая одинаковые оценки при каждом заходе.

## Связи

Зависит от `web/src/ui.tsx` (`PageHead`, `Chip`, `NexAsk`, `useApp`) и `web/src/data.ts` (`groups`, `students`, `subjectsByGroup`, `gradesFor`, `scheduleDays`, `scheduleSlots`). Три экспортируемые функции (`Schedule`, `Journal`, `Attendance`) подключаются как отдельные маршруты из `App.tsx`; логически соседствует с `web/src/pages/academic-beta.tsx` (дисциплины, ведомости, приказы — та же предметная область, но на бета-инфраструктуре).
