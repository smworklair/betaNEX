# web/src/pages/accounting.tsx

Финансовый (бухгалтерский) раздел — самый большой набор экранов среди страниц: обзор-«кокпит», платежи, задолженности, начисления, калькуляторы рассрочки, зарплата, стипендии, бюджет, отчёты. Смешивает два подхода: большинство экранов — витрины над статичными моковыми данными из `data.ts`, а «Стипендии» — уже полноценный CRUD на бета-инфраструктуре (`beta/store`+`beta/manager`), как в `academic-beta.tsx`.

## Ключевое

- `rub(n)`, `krub(n)` — форматирование сумм в рубли (`rub` — полная сумма «₽ 12 345», `krub` — сокращённая «₽ 12K»).
- `FinNote` — общая карточка-подсказка NEX для финансовых экранов (текст + кнопка «Разобрать»), с вариантом акцента `tone: 'warn'`.
- `FinOverview` — финансовый «кокпит»: переключатель периода (месяц/квартал/год), сгенерированные на лету цифры прогноза денежного потока, KPI-плашки, список «что требует решения» с возможностью локально отметить пункт выполненным (`done`, только в React-состоянии), графики ликвидности и структуры доходов, дебиторка по срокам просрочки, чек-лист закрытия периода.
- `FinPayments` — таблица-реестр всех платежей (`finance.payments`).
- `FinDebts` — список должников (`charges`, отфильтрованные по `!paid`) с кнопками «напомнить» (`toast`).
- `FinCharges` — таблица всех начислений (оплаченных и нет).
- `FinCalc` — два калькулятора: рассрочка (сумма/срок/пеня — интерактивные поля) и прогноз поступлений (статичные сценарии).
- `FinPayroll` — ведомость зарплаты (`payroll`) с итоговой суммой фонда.
- `FinScholarship` — единственный CRUD-экран раздела: стипендии через `useCollection<Scholarship>('fin-scholarships', ...)` и `EntityManager`, с сид-данными из `finance.scholarships`.
- `FinBudget` — план/факт бюджета по статьям (`budgetLines`) с индикатором перерасхода.
- `FinReports` — список готовых/готовящихся отчётов (`reports`) с кнопкой скачивания (`toast`-заглушка).
- `FIN_ICONS` — набор иконок раздела, переэкспортируется для использования в меню `App.tsx`.

## Как это работает

Большая часть цифр в `FinOverview` (денежный поток, прогноз, обязательства недели) — не реальные вычисления от базы, а формулы поверх констант и синусоиды (`Math.sin`) для правдоподобного графика — то есть демонстрационная имитация «живой» аналитики. Действия («Сверить банк», «Напомнить всем», «Скачать отчёт» и т.д.) почти везде — это `toast(...)`, не меняющие реальные данные, за исключением `FinOverview`, где отметка задачи выполненной (`complete`) действительно скрывает её из локального списка `incomplete`, и `FinScholarship`, где `EntityManager` даёт настоящий CRUD поверх localStorage.

## Связи

Зависит от `web/src/ui.tsx` (`PageHead`, `Chip`, `NexAsk`, `Beta`, `useApp`), `web/src/charts.tsx` (`Donut`, `Bars`, `Legend`, `Line`, `Segment`), `web/src/data.ts` (`finance`, `charges`, `payroll`, `budgetLines`, `reports`, `students`), `web/src/beta/store.ts` (`useCollection`, `Entity` — только для стипендий) и `web/src/beta/manager.tsx` (`EntityManager` — только для стипендий). Девять экспортируемых функций подключаются как отдельные маршруты финансового раздела в `App.tsx`.
