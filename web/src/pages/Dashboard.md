# web/src/pages/Dashboard.tsx

Экран «Дашборд» (страница безопасности/системного мониторинга). В отличие от `CommandCenter.tsx` (аналитический центр принятия решений) и `Home.tsx` (личный главный экран), этот экран фокусируется на состоянии самой системы: сессии, аудит, неудачные входы, целостность сервисов — то есть смотрит на приложение «снаружи внутрь», как консоль безопасности/эксплуатации.

## Ключевое

- `Dashboard` (default export) — точка входа: для роли `admin` показывает `SecurityConsole`, для остальных — упрощённый `FocusDashboard`.
- `SecurityConsole` — полная консоль безопасности: статус-баннер (защищена/требует внимания), 4 KPI, лента аудита, критические действия, активные сессии, неудачные попытки входа со спарклайном тренда, панель мониторинга ИИ (`aiInsights` с процентом уверенности в виде кольца), плитки состояния сервисов (`services`), важные уведомления.
- `FocusDashboard` — облегчённая версия для не-админов: приветствие, 3 KPI (пары сегодня, требующие действий, количество групп), короткий список «требует внимания» из первых 3 уведомлений.
- `Kpi` — локальная карточка метрики с трендом (такая же по смыслу, как в `CommandCenter.tsx`, но реализована отдельно — код не переиспользуется между файлами).

## Как это работает

Все данные — статические моковые константы из `data.ts` (`sessions`, `auditEvents`, `failedLogins`, `notifications`, `aiInsights`, `services`, `criticalActions`). Единственная динамика на странице — вычисление `flagged` (число подозрительных попыток входа) для выбора цвета/текста статус-баннера. Кнопки вроде «Завершить сессию» вызывают `toast(...)` без реальной мутации данных — это витрина UI, а не рабочая функциональность. `NexAsk` из `ui.tsx` встроен в несколько карточек как быстрый способ спросить NEX об этом конкретном блоке данных.

## Связи

Зависит от `web/src/ui.tsx` (`useApp`, `Chip`, `Avatar`, `Sparkline`, `NexAsk`, `severityTone`, `severityLabel`) и `web/src/data.ts` (много моковых источников: `sessions`, `auditEvents`, `failedLogins`, `notifications`, `aiInsights`, `services`, `criticalActions`, `groups`, `roleLabel`). Самостоятельный экран, маршрутизируемый из `App.tsx`; логически близок к `web/src/pages/CommandCenter.tsx`, но не переиспользует с ним компоненты — оба реализуют похожие KPI-карточки независимо.
