# web/src/pages/agents.tsx

Экран «Агенты» — экспериментальный центр управления автономными ИИ-агентами NEX, доступный только администратору. Ключевая идея, прямо заявленная в комментарии файла: человек управляет «политикой», а не отдельными кнопками — задаёт уровень автономии, пишет правила словами, разбирает очередь подтверждений с прогнозом последствий, вместо того чтобы вручную запускать каждое действие.

## Ключевое

- `Agents` (default export) — весь экран: блок автопилота, поручение задачи, очередь подтверждений, правила, сетка карточек агентов, журнал автономных действий.
- `StepsRun` — визуализация выполнения плана по шагам (тикает по таймеру), переиспользуется и для делегированных задач, и для подтверждённых действий из очереди; помечена комментарием `BACKEND: стрим статуса воркера` — то есть в реальности это будет живой стрим статуса, а не таймер.
- `Delegate` — блок «Поручить NEX»: пользователь описывает задачу словами, NEX (реальный LLM через `llmPlan`, если подключён, иначе шаблонный `planFor`) составляет план из шагов, пользователь утверждает или отменяет, затем план выполняется через `StepsRun`.
- `Queue` — очередь действий, ожидающих подтверждения человека (`PendingAction[]`), с кнопкой «Прогноз» (`dryRun(action)` — показывает, что изменится, до принятия решения), «Подтвердить» (запускает `StepsRun`) и «Отклонить».
- `Rules` — список автоматизаций, описанных обычным языком («если …, то …»); при добавлении нового правила агент подбирается эвристикой `guessAgent(text)` — помечено как то, что в реальности разбор текста возьмёт на себя LLM.
- `AgentCard` — карточка одного агента: название, домен, описание, пауза/возобновление, переключатель уровня автономии (`AUTONOMY_LEVELS`), последнее действие и счётчик действий за день.
- Локальное состояние `levels` в `Agents` — общий уровень автономии всех агентов; `applyPreset` меняет его разом для всех через выбор пресета автопилота (`AUTOPILOT_PRESETS`).

## Как это работает

Экран собирает несколько независимых кусков UI вокруг общих данных из `web/src/agents.ts` (константы `AUTONOMY_LEVELS`, `AUTOPILOT_PRESETS`, `RULES`, функции `agentById`, `guessAgent`, `dryRun`) и `web/src/api/agents.ts` (`AGENTS`, `QUEUE`, `AGENT_LOG` — сейчас моки, но уже вынесены в отдельный `api/`-слой специально, чтобы позже заменить на реальный `/api/v1/agents/*` правкой одного файла, как явно указано в комментарии). `Delegate` — единственное место на этом экране, где может использоваться настоящий LLM (`llmReady()`/`llmPlan`) для составления плана; в остальном вся «работа» агентов имитируется таймерами (`StepsRun`) и обновлением локального React-состояния (`toast`, удаление из очереди), без реальных побочных эффектов.

## Связи

Зависит от `web/src/ui.tsx` (`PageHead`, `Chip`, `Beta`, `useApp`), `web/src/features/ai/nexbrain.ts` (`planFor` — тот же генератор шагов плана, что используется в `Chat.tsx`), `web/src/features/ai/llm.ts` (`llmReady`, `llmPlan`), `web/src/agents.ts` (не входит в текущий список файлов — модель уровней автономии, пресетов, правил, dry-run) и `web/src/api/agents.ts` (мок-слой данных агентов, готовый к замене на реальный backend). Единственный экран раздела «Агенты», маршрутизируется из `App.tsx`; логически перекликается с `AgentRun` в `web/src/pages/Chat.tsx` (тот же паттерн «видимое пошаговое выполнение»), но не переиспользует код напрямую.
