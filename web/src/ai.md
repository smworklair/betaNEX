# web/src/ai.tsx

Слой «ИИ везде» поверх интерфейса NEX: проактивная подсказка сверху страницы, встроенная в поток страницы ИИ-панель (открывается прямо под нажатым блоком) и всплывающее меню действий по выделенному тексту. Это UI-обвязка над двумя источниками ответов — реальным LLM (Gemini через `./llm`) и локальным моком-ответчиком (`./nexbrain`), которые взаимозаменяемы через фолбэк.

## Ключевое

- `ProactiveStrip()` — полоска над контентом страницы с проактивной подсказкой NEX (`pageInsight(page)` из `nexbrain`), которую можно скрыть на текущей странице; управляется настройкой `prefs.strip`.
- `InlinePanel()` / `InlinePanelHost()` — чат-панель, портализуемая (`createPortal`) прямо в DOM-узел, куда кликнул пользователь (`inlineHost` из контекста приложения); ведёт диалог с контекстом текущей страницы/выбранного студента (`buildContext`).
- `SelectionPopover()` — плавающее меню, появляющееся при выделении текста мышью: «Спросить NEX», «Передать ИИ», «Резюме», «Перевести», «Объяснить», плюс быстрые действия «В задачу/заметку/календарь/сообщение/письмо» и «Копировать».
- `SelExplain()` — маленькая контекст-независимая всплывающая подсказка-объяснение выделенного термина (через LLM, либо эвристику `explainSelection` как фолбэк).
- `explainSelection(text)` — эвристика без ИИ: по regex-паттернам угадывает тип выделенного текста (процент, сумма, ФИО, код группы) и даёт заготовленное объяснение.
- `AiLayer()` — собирает три плавающих ИИ-поверхности (`SelectionPopover`, `SelExplain`, `InlinePanelHost`) в одном компоненте, монтируемом один раз в каркасе приложения.

## Как это работает

Везде используется один и тот же паттерн: сначала пробуем реальный LLM (`llmReady()` + `llmAsk(...)`), при ошибке или отсутствии ключа — откатываемся на локальный мок (`nexReply` из `nexbrain` для диалога, `explainSelection` для объяснений). `InlinePanel` и `SelExplain` держат контекст текущего экрана (`buildContext`), чтобы вопрос в LLM уходил вместе с описанием того, что видит пользователь (студент, страница). `SelectionPopover` слушает `mouseup`/`mousedown` на всём документе и игнорирует выделения внутри уже открытых ИИ-поверхностей и полей ввода, чтобы не конфликтовать сама с собой.

## Связи

Зависит от контекста приложения `useApp`/`Chip`/`NexAsk` (`./ui`), демо-данных (`./data`), локального «мозга»-заглушки (`./nexbrain`), обёртки над LLM (`./llm`), рендерера markdown (`./md`) и клиентского локального хранилища сущностей (`./beta/store` — `useCollection`, для создания задач/заметок/событий из выделения). Монтируется в `App.tsx` (`<AiLayer />`) и переиспользуется в `MobileShell`/`Shell`.

## На что обратить внимание

`InlinePanel` и `SelExplain` не хранят историю на сервере — это фронтенд-only функциональность прототипа; реальные «действия» (создание задачи из выделения через `createTask`/`createNote`/`createEvent`) пишут в локальное browser-хранилище через `useCollection`, а не на бэкенд.
