# web/src/blocks.tsx

Небольшой файл с готовыми составными UI-блоками для раздела «студенты»: список студентов из группы риска и выезжающая сбоку карточка (drawer) с подробностями по конкретному студенту. Логика и данные здесь всё ещё моковые (тестовый набор `data.ts`), но компоненты уже написаны так, будто работают с реальными расчётами — это переиспользуемые куски, которые вставляются в разные экраны (например, в командный центр и в карточку студента).

## Ключевое

- `attendanceRate(id)` — детерминированная псевдослучайная посещаемость по id студента (заглушка вместо реального расчёта).
- `avgGrade(id, group)` — средний балл студента по оценкам из `gradesFor(group)` (нулевые оценки-пропуски исключаются).
- `groupAvg(group)` — средний балл по всей группе.
- `hasDebt(lastname)` — есть ли непогашенная задолженность по оплате (ищет в `finance.payments` по совпадению фамилии).
- `AtRiskList` — компонент-таблица студентов «в зоне риска» (низкая посещаемость или средний балл), с кликом по строке, открывающим карточку студента.
- `ContextDrawer` — выезжающая справа панель с деталями студента: сводка от NEX, статистика, история событий, быстрые действия (сформировать справку, открыть журнал, поставить задачу куратору).

## Как это работает

Оба компонента читают глобальное состояние приложения через `useApp()` из `web/src/ui.tsx`: `AtRiskList` использует `openStudent` для перехода в контекст студента по клику на строку, `ContextDrawer` читает `objStudent` (id открытого студента) и рендерится только когда он задан (`null` — компонент ничего не выводит). Данные о студентах и оценках берутся напрямую из моковых массивов `students`, `finance`, `gradesFor` в `data.ts` — здесь нет обращения к серверу. Внутри `ContextDrawer` есть блок `NexAsk`, который отправляет предзаполненный вопрос ИИ-помощнику NEX с контекстом конкретного студента.

## Связи

Зависит от `web/src/ui.tsx` (`useApp`, `Chip`, `NexAsk`) и `web/src/data.ts` (моковые данные `students`, `finance`, `gradesFor`). Используется другими экранами, где нужен виджет риска или карточка студента — в частности связан с командным центром (см. `web/src/pages/CommandCenter.tsx`) и общим механизмом открытия объектов через `openStudent`/`objStudent`.
