# web/src/llm.ts

Единая точка входа к ИИ для фронтенда NEX. Браузер НЕ обращается к `ai-gateway/` (Python, см. `ai-gateway/README.md`) напрямую — только к своему же бэкенду `nexd`, тем же origin и той же cookie-сессией, что и весь остальной `/api/v1/*` (см. `web/src/api/client.ts`). `nexd` сам проксирует `/api/v1/ai/*` в `ai-gateway` по внутренней сети (`internal/platform/httpapi/aiproxy.go`), подставляя `tenant_id` из аутентифицированной сессии и подписывая запрос секретом, общим с `ai-gateway`. Так `X-Tenant-Id` перестаёт быть самопредставлением клиента: раньше браузер бил в `ai-gateway` напрямую и мог вписать в этот заголовок чужого тенанта, обходя его бюджет. Ключи провайдеров (Gemini/OpenAI/DeepSeek/...) по-прежнему живут только в переменных окружения `ai-gateway`. Конфигурация — переменная `VITE_AI_ENABLED` (см. `.env.example`), это ФЛАГ, а не URL: пусто — демо-режим без сети, `1`/`true` — реальные вызовы через `nexd`.

## Ключевое

- `AI_GATEWAY_BASE` — тот же `API_BASE`, что и у остального API (`web/src/api/client.ts`): ИИ-запросы идут на тот же origin, что и `nexd`.
- `AI_GATEWAY_CONFIGURED` — признак, что ИИ включён (`VITE_AI_ENABLED=1`/`true`), не зависит от того, доступен ли реально `nexd`/`ai-gateway`.
- `LlmProvider` — тип имени провайдера на шлюзе (`'gemini' | 'custom' | 'openai' | 'deepseek' | 'qwen' | 'kimi' | 'gigachat' | 'yandexgpt'`).
- `getProvider`/`setProvider` — только ПРЕДПОЧТЕНИЕ провайдера (имя, без ключей) в localStorage; пусто — шлюз берёт провайдера по умолчанию из своего конфига.
- `llmReady()` — готов ли ИИ отвечать, то есть включён ли флаг (ключи больше не проверяются — их просто нет на клиенте).
- `ORG_CONTEXT` — большой системный промпт главного чата NEX (`Chat.tsx`): личность, режимы ответа, демо-данные организации. Передаётся как явный `system`-оверрайд — в отличие от мини-чатов на страницах (`AiBox`/`InlinePanel`), которые вместо этого шлют `context` и получают промпт раздела с сервера.
- `PageContext` — контекст страницы `{ page, title?, facts?, state? }`: `page` превращается на сервере в ролевую инструкцию (см. `ai-gateway/app/core/context_registry.py`), `facts`/`state` — живые данные экрана.
- `llmAsk(user, opts)` — единая точка входа: `POST {AI_GATEWAY_BASE}/api/v1/ai/ask` с `credentials: 'include'` (сессия `nexd` — прокси требует аутентифицированного актора) и телом `{message, history, system?, context?, provider?}`, бросает исключение при ошибке/не-конфигурации.
- `fetchProviders()` — список провайдеров, реально настроенных на сервере (`GET /api/v1/ai/providers`, с сессией), для выбора в Настройках.
- `checkGateway()` — жив ли шлюз (`GET /api/v1/ai/healthz`, с сессией — прокси `nexd` переписывает путь в `/healthz` на стороне `ai-gateway`) — для индикатора статуса в Настройках.
- `llmPlan(task)` — просит модель составить план из 3-5 шагов для задачи («Поручить NEX»), парсит ответ построчно.

## Как это работает

`llmAsk` собирает тело запроса (историю переводит в формат `{role: 'user'|'assistant', content}`, добавляет `context`/`system`/`provider`, если заданы) и шлёт один `fetch` в `nexd` с таймаутом через `withTimeout` (AbortController) и cookie-сессией. При ошибке (ИИ не включён, нет сессии, не-2xx ответ, пустой текст) выбрасывается исключение — вызывающий код обязан откатиться на локальный мок-слой (`nexbrain.ts`), этот файл фолбэком сам не занимается.

## Связи

Зависит от `web/src/api/client.ts` (`API_BASE` — тот же origin, что и у REST API). Используется компонентами, которые обращаются к ИИ-помощнику NEX: `web/src/nexbrain.ts` (локальный фолбэк), `web/src/pages/Settings.tsx` (выбор провайдера из списка сервера + статус шлюза), `web/src/pages/aibox.tsx` и `web/src/ai.tsx` (мини-чаты с контекстом раздела и своей историей), `web/src/pages/Chat.tsx`, `web/src/terminal.tsx`, `web/src/pages/Home.tsx`.

## На что обратить внимание

`ORG_CONTEXT` по-прежнему содержит захардкоженные демо-данные организации — заглушка для прототипа, в реальной системе должна замениться на RAG по настоящей БД (см. `docs/ai/README.md`, §2.6). Подробный разбор архитектуры (что сегодня, что в планах) — там же, `docs/ai/README.md`.
