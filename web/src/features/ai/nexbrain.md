# web/src/features/ai/nexbrain.ts

Локальный «мозг» ИИ-помощника NEX — детерминированный мок-движок, который отвечает на вопросы пользователя по ключевым словам, без обращения к настоящей LLM. Используется как основной источник ответов для демонстрации (или как фолбэк, когда реальный LLM-провайдер из `llm.ts` недоступен или не настроен). Файл прямо описан в комментарии как «одна интеллектуальность, общая для узкой контекстной панели (Cmd+E) и полноценной страницы чата» — то есть оба места используют один и тот же движок вместо дублирования логики. Важный принцип: «чистые данные на входе/выходе — UI решает, как это отрисовать» (сам файл не рендерит JSX).

## Ключевое

- `attendanceRate`, `avgGrade`, `groupAvg` — те же расчётные функции, что и в `blocks.tsx` (посещаемость, средний балл студента/группы) — дублирование на основе одних и тех же моковых данных.
- `atRisk()` — список студентов в зоне риска (низкая посещаемость/балл), возвращает `RiskRow[]`.
- `NexData` — тип «блока данных», который чат может отрисовать под текстовым ответом (`'atrisk' | 'finance' | 'kpi' | 'security'`).
- `NexReply` — форма ответа NEX: текст, опциональная навигация (`nav`), опциональное подтверждаемое действие (`action`), опциональный блок данных (`data`).
- `NexCtx` — контекст запроса: какой студент открыт, какая страница активна, есть ли выделенный текст (`seed`).
- `PAGE_TITLES` — человекочитаемые названия разделов по их id.
- `pageInsight(page)` / `PAGE_INSIGHTS` — проактивная подсказка NEX для конкретного экрана («NEX говорит первым») плюс готовые вопросы-чипы.
- `planFor(label)` — генерирует список шагов «агентного» выполнения задачи по ключевым словам в её названии.
- `nexReply(q, ctx)` — главная функция: по тексту вопроса и контексту подбирает готовый ответ через последовательность регулярных выражений.

## Как это работает

`nexReply` — это цепочка `if`/`regex` проверок по убыванию специфичности: сначала обрабатывается узкий контекст (выделенный текст `seed`, открытый студент `student`), затем более общие темы организации (безопасность, финансы, посещаемость, расписание, кампус, агенты и т.д.), и в конце — общий fallback-ответ. Ни одна ветка не обращается к сети — все ответы строятся из статических шаблонов строк и моковых данных (`data.ts`), поэтому результат полностью предсказуем и мгновенен.

## Связи

Зависит от `web/src/data.ts` (`students`, `finance`, `gradesFor`). Используется `web/src/features/ai/nexdata.tsx` (рендер блоков `NexData`) и, ожидаемо, `web/src/pages/Chat.tsx`, `web/src/pages/Home.tsx` (терминал) как источник ответов, когда реальный LLM (`web/src/features/ai/llm.ts`) недоступен или явно не используется.

## На что обратить внимание

Логика расчётов (`attendanceRate`, `avgGrade`, `groupAvg`, проверка задолженности) дублирует то, что уже есть в `web/src/blocks.tsx` — это не общий модуль, а два независимых места с одинаковыми формулами поверх одних моковых данных.
