# web/src/features/terminal/terminal.tsx

«NEX Терминал» (альфа) — командная консоль администратора, дающая доступ ко всей функциональности сайта через человекочитаемые команды вместо навигации по разделам. Важно понимать позиционирование: это **не** замена обычного UI (страницы `pages/*` продолжают жить как жили) и **не** инструмент для программиста — команды пишутся русскими/английскими словами («финансы», «риск», «новая задача Проверить отчёт»), подсказки всегда на виду, а результат отображается теми же виджетами (таблицы, KPI-плитки, графики), что и на обычных страницах. Файл был недавно полностью переписан: раньше терминал существовал в нескольких разрозненных реализациях, теперь один реестр команд и один компонент меню (`CmdMenu`) обслуживают и компактный блок на «Главном», и полноэкранную среду (`TerminalWorkspace`).

## Модель команд

Ядро файла — статический реестр `TERM_COMMANDS: TermCommand[]`. Каждая команда (`TermCommand`) описывается:

- `id` — каноническое написание, которое видно в подсказках (например `'долги'`);
- `domain` — один из восьми доменов (`Обзор`, `Аналитика`, `Финансы`, `Безопасность`, `Задачи`, `Кампус`, `Люди`, `Расписание`) — используется для группировки в меню и для доменных чипов в полноэкранной среде;
- `aliases` — список слов (рус/англ), по первому слову которых команда распознаётся (`['долги', 'debts', 'должники', 'задолженность']`);
- `arg` — необязательная подпись аргумента для подсказки (например `'<текст>'`);
- `mutates` — флаг «команда меняет данные» (используется для визуальной пометки в меню, класс `mutate`);
- `run(args, ctx)` — функция-обработчик, возвращающая `TermRes` (структурированный результат: `kind: 'text' | 'table' | 'kpi'`, плюс необязательный `node` — живой React-узел, например график).

Команды охватывают все ключевые домены: сводка системы (`обзор`), аналитика и риск-студенты (`аналитика`, `риск`, `группы`, `посещаемость`, `студент`), финансы (`финансы`, `долги`, `платежи`, `стипендии`, `напомнить должникам`), безопасность (`безопасность`, `входы`, `сессии`, `сервисы`, `аудит`, `люди`), задачи (`задачи`, `новая задача`, `готово`, `уведомить`) и расписание/кампус (`расписание`, `экзамены`, `начисления`, `сотрудники`, `новое окно`). Часть команд мутирует состояние — например `новая задача` и `готово` напрямую читают и пишут в общую коллекцию задач `'tasks'` через переданный в `EngineCtx` объект `tasks` (`{ items, add, update }`), то есть терминал и страница `pages/tasks.tsx` работают с одними и теми же данными, а не с копиями.

Диспетчеризация команды по введённой строке — функция `execRegistry(line, ctx)`: берёт первое слово строки, ищет команду по `aliases`, если находит — вызывает её `run()` с оставшимися словами как `args`; если нет — возвращает `null`, и вызывающий код (в `TerminalWorkspace`) либо пробует бэкенд, либо показывает «не понял команду».

### Ранжирование поиска: `searchCommands` / `cmdScore`

Для меню команд есть отдельный слой поиска, не завязанный на строгое совпадение первого слова:

- `cmdScore(c, q)` — считает «ранг» совпадения команды `c` с запросом `q` (уже в нижнем регистре): `0` — точное совпадение id или алиаса, `1` — id начинается с `q`, `2` — какой-то алиас начинается с `q`, `3` — `q` входит в id, `4` — `q` входит в описание (`desc`); `-1` — не совпало вовсе.
- `searchCommands(input)` возвращает `CmdSearch` — если строка пустая, отдаёт все команды, сгруппированные по домену (`paletteByDomain()`, отсортированные по `id`); если непустая — единый плоский список без секций (`domain: null`), отсортированный по `cmdScore` (лучшие совпадения сверху). Поле `flat` дублирует список без вложенности групп — используется для управления выделением стрелками и для `Tab`/`Enter`.

## Компонент `CmdMenu`

`CmdMenu({ search, sel, onHover, onPick, up })` — единственная реализация выпадающего меню команд (`.cmenu`), используемая и в компактном блоке терминала на «Главном», и в полноэкранной среде `TerminalWorkspace`. Раньше существовали отдельные модальные палитры для каждого режима — сейчас ввод всегда остаётся в основной строке ввода родителя, а меню лишь фильтрует и подсвечивает совпадения по этому вводу. Особенности:

- Управление полностью делегировано родителю: сам `CmdMenu` не хранит выделение (`sel` передаётся снаружи), а лишь рендерит список и сообщает о наведении (`onHover`) и выборе (`onPick`).
- Каждый пункт — `<button>` с `onMouseDown={(e) => { e.preventDefault(); onPick(c); }}` — `preventDefault` на `mousedown` нужен, чтобы клик мышью по пункту меню не забирал фокус из текстового поля ввода (иначе следующее нажатие клавиши не попадёт в input).
- Выбранный пункт всегда виден в списке — реализовано через `ref`-коллбэк, вызывающий `scrollIntoView({ block: 'nearest' })` именно на элементе с `idx === sel`.
- Проп `up` переворачивает меню вверх (`.cmenu.up`) — используется для компактного блока на «Главном», где меню физически не помещается вниз.
- Внизу меню — статичная подсказка по горячим клавишам (`↑↓` выбор, `Tab` подставить, `Enter` выполнить, `Esc` закрыть).

## `TerminalWorkspace` — полноэкранная среда

`TerminalWorkspace({ ctx, remote, onClose })` — полноэкранная «командная среда» (модель похожа на Raycast): командная строка закреплена сверху, под ней ряд доменных чипов, а ниже — **лента результатов блоками** (`tws-feed`), где каждая новая команда добавляется в начало списка (`setBlocks((b) => [{ id, cmd, pending: true }, ...b])`), а не в конец — благодаря этому свежий ответ появляется прямо под строкой ввода, и не нужно скроллить вниз, чтобы его увидеть. Рендерится через `createPortal(..., document.body)`, потому что карточки сайта используют `backdrop-filter`, из-за чего `position: fixed` внутри них вёл бы себя как `absolute`.

Ключевые механизмы:

- **Блок результата** (`WsBlock`) сначала добавляется со статусом `pending: true` (анимация «выполняю…» с мигающим курсором), затем асинхронно дозаполняется (`finish()`) результатом и меткой времени выполнения (`stamp()` — миллисекунды + источник: `демо` или `сервер`).
- **Гибридный режим бэкенд/демо**: если передан `remote` (функция `POST /terminal/exec`) и первое слово команды входит в `TERMINAL_BACKEND_TOKENS` (список команд, которые умеет обслуживать реальный сервер), запрос уходит на бэкенд; при ошибке — тихий фолбэк на локальный `execRegistry(cmd, ctx)` с честной пометкой в `stamp()`, что сервер был недоступен. Команды не из этого списка выполняются локально через демо-движок `execRegistry`.
- **Меню (`menuVisible`)**: при пустом вводе меню появляется только по явному нажатию `/` (`setMenuOpen(true)`); при непустом — открывается автоматически, если есть хоть одно совпадение (`search.flat.length > 0`). Так пустая среда не заваливает пользователя списком команд сразу, но при наборе текста подсказки появляются мгновенно.
- **Выбор из меню** (`pick`): если у команды есть `arg` (например `студент <фамилия|email>`), она не выполняется сразу — вместо этого строка ввода заполняется `id + ' '` и фокус остаётся в поле, чтобы пользователь дописал аргумент; если аргумента нет — команда выполняется немедленно (`runLine(c.id)`).
- **История команд**: при закрытом меню стрелки `↑`/`↓` листают ранее введённые команды (`hist`, до 40 последних), как в обычной shell-консоли; при открытом меню те же стрелки двигают выделение по списку совпадений — то есть одни и те же клавиши переключают поведение в зависимости от состояния `menuVisible`.
- **Esc двухступенчатый**: сначала закрывает открытое меню (если оно открыто — `menuRef.current`, актуализируется на каждый рендер через ref, чтобы обработчик на `window` видел свежее значение без пересоздания подписки), и только повторным нажатием (когда меню уже закрыто) закрывает всю полноэкранную среду через `onClose`.
- **Автозапуск**: при первом входе в среду автоматически выполняется команда `обзор` (защёлка `booted` через `useRef`, чтобы не сработать дважды при `StrictMode`-повторном монтировании в разработке), и фокус сразу ставится в строку ввода.
- **Доменные чипы** (`tws-domains`) — быстрые ярлыки по восьми доменам, каждый привязан к одной «типовой» команде домена (`DOMAIN_META`); клик выполняет `runLine(d.cmd)`, при этом тоже используется `mousedown + preventDefault`, чтобы фокус не покидал строку ввода.
- Каждый блок результата в ленте можно перезапустить кнопкой «Выполнить ещё раз» (`RotateCw`) — вызывает `runLine(b.cmd)` заново, создавая новый блок сверху (старый остаётся в ленте).
- Спецкоманда `очистить`/`clear` не идёт через реестр команд — обрабатывается отдельной веткой в начале `runLine`, полностью очищая ленту (`setBlocks([])`).

## Рендер результата: `TermResBlock`

`TermResBlock({ r })` — единая функция рендера структурированного ответа `TermRes`, используемая и в компактном блоке, и в ленте `TerminalWorkspace`. Показывает заголовок (`r.title`), текст как markdown (`<Md text={r.text} />`), KPI-плитки (`r.kind === 'kpi'`), таблицу (`r.kind === 'table'`), произвольный React-узел с графиком (`r.node`, например `<Line>`/`<Donut>` из `../charts`) и подсказку-хинт с рекомендуемыми следующими командами (`r.hint`).

## Клавиатурные взаимодействия (сводно)

| Клавиша | Меню закрыто | Меню открыто |
|---|---|---|
| `/` (пустой ввод) | открывает меню | — |
| `↑` / `↓` | листает историю команд | двигает выделение по списку |
| `Tab` | — | подставляет `id [arg ]` выбранной команды в строку |
| `Enter` | выполняет введённую строку | выполняет выбранную команду (или строку как есть, если ничего не выбрано) |
| `Esc` | закрывает всю среду (`onClose`) | закрывает только меню |

## Связи

Зависит от `../../ui` (`useApp`), `../../components/md` (`Md` — рендер markdown в ответах), `../../components/charts` (`Line`, `Donut`, `Legend`, `Segment`), большого набора моковых данных из `../../data` (`students`, `staff`, `sessions`, `auditEvents`, `failedLogins`, `services`, `finance`, `groups`, `scheduleDays`, `scheduleSlots`, `charges`, `exams`), вычисляемых метрик из `../ai/nexbrain` (`atRisk`, `attendanceRate`, `avgGrade`, `groupAvg`, `PAGE_TITLES`), бэкенд-интеграции `../../api/terminal` (`TERMINAL_BACKEND_TOKENS`, `TermResult`) и типа `Entity` из `../../beta/store`. Коллекция задач `'tasks'`, с которой оперируют команды `новая задача`/`готово`, — та же самая, что использует `pages/tasks.tsx` — это осознанно общая «экосистема» данных, а не изолированная копия.

## На что обратить внимание

Терминал и полноэкранная среда явно не заменяют навигацию сайта — команда `раздел`/`open`/`открой` (`ctx.setPage(hit.page)`) лишь переключает на обычный маршрут внутри того же приложения. Значит любые новые страницы должны быть отражены в `PAGE_TITLES` (`../ai/nexbrain`), иначе команда `раздел X` их не найдёт.
