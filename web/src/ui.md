# web/src/ui.tsx

Центральный модуль общей инфраструктуры фронтенда: глобальный контекст приложения (`AppProvider`/`useApp`) и набор мелких переиспользуемых UI-примитивов (чипы, шапка страницы, аватар, спарклайн, карточка ИИ-инсайта, «спроси NEX»-кнопки). Практически каждая страница приложения (`pages/*`, `terminal.tsx`, `stream.tsx`) импортирует что-то отсюда — это ядро, вокруг которого собран весь фронтенд.

## Ключевое

- `AppProvider` / `useApp()` — React-контекст с глобальным состоянием: текущий пользователь (`user`), тема (`theme`), текущий раздел навигации (`page`/`setPage`, за кулисами — URL через `wouter`), открытая карточка студента (`objStudent`), состояние командной палитры (`cmdOpen`), «инлайн»-панель NEX внутри страницы (`inlineHost`/`openInlineAt`/`closeInline`), настройки персонализации (`prefs`/`setPref`), полноценный чат NEX (`chatLog`/`openChat`), тосты (`toast`) и многое другое навигационное/UI-состояние.
- `Prefs` (интерфейс) и `DEFAULT_PREFS` — настройки персонализации интерфейса, персистентные в `localStorage` (`nex-prefs`): акцентный цвет, плотность, шрифт, скругления углов, автоскрытие боковой панели, набор кнопок докбара/топбара, конструктор главного экрана.
- `useIsMobile()` — хук, возвращающий `true` на экранах ≤860px, реагирует на `matchMedia`-изменения; используется всеми адаптивными разделами (`beta.tsx` Messenger, `social.tsx` Mail и др.).
- `NexAsk({ q, label, subtle })` — маленькая кнопка «Спросить NEX», встраиваемая в карточки/таблицы/графики по всему приложению; при клике находит ближайший подходящий контейнер (`.nex-strip, .ai-card, .card, .fade`) и открывает там инлайн-ответ через `openInlineAt`.
- `AskHero()` — крупный блок «спросите NEX» с быстрыми интентами-чипами (сводка дня, зона риска, финансы, безопасность), открывающий полноценный чат (`openChat`).
- `Chip`, `Avatar`, `PageHead`, `Sparkline`, `AiInsightCard`, `Soon`, `Beta` — набор мелких презентационных компонентов: цветной статус-чип, аватар с инициалами, заголовок страницы с сабтайтлом и экшенами, мини-график-спарклайн (чистый SVG без библиотек), карточка ИИ-инсайта с индикатором уверенности, бейджи «в разработке»/«beta».

## Как это работает

`AppProvider` оборачивает всё приложение и хранит десятки кусков состояния через отдельные `useState`, синхронизируя часть из них с `localStorage` (тема, демо-пользователь, персонализация, включена ли боковая панель/пульс). Текущий раздел (`page`) — не `useState`, а производная от `wouter`-хука `useLocation()`: `page = location.replace(/^\//, '') || 'home'`, `setPage(p)` вызывает `navigate('/' + p)`. Это даёт настоящий deep-linking (URL раздела можно скопировать и открыть заново — SPA-фолбэк уже настроен в `nginx.conf`/`vercel.json`) без изменения контракта `useApp().page`/`setPage` для остального кода. При старте есть попытка восстановить настоящую сессию бэкенда через `apiMe()` (модуль `./api/auth`) — если сервер сконфигурирован и есть валидная cookie, пользователь логинится автоматически; если бэкенда нет, используется демо-пользователь из `localStorage` (`nex-demo-user`). Персонализация (`prefs`) применяется не через инлайн-стили, а через `data-*`-атрибуты на `<html>` (`data-accent`, `data-density` и т.д.) — вся визуальная логика этих настроек живёт в CSS, а не в JS.

## Связи

Зависит от `wouter` (`useLocation`), `./data` (типы `Role`, `Severity`), `./nexbrain` (тип `NexReply`), `./api/auth` (`apiMe`, `authConfigured`, `primaryRole`) и `./home` (`DEFAULT_HOME_BLOCKS`, `DEFAULT_HOME_SHORTCUTS`). Практически весь фронтенд — обратная зависимость: все файлы из `pages/*`, а также `terminal.tsx` и `stream.tsx`, импортируют `useApp`, `PageHead`, `Chip`, `NexAsk` и другие экспорты этого модуля.

## На что обратить внимание

`inlineHost`/`openInlineAt`/`closeInline` — не про модалки, а про открытие мини-чата NEX прямо в потоке страницы (карточка «раздувается» на месте клика, а не поверх экрана). При переключении хоста предыдущий контейнер аккуратно очищается от CSS-классов (`inline-host`, `inline-row`) перед назначением новых — если добавлять новую точку входа для этой функциональности, важно сохранить эту симметрию, иначе на странице останутся «залипшие» раздутые карточки.
