# web/src/api/agents.ts

API-обёртка для будущего (пока не существующего на бэкенде) модуля «Агенты» — единственный файл в `api/`, который не обращается к серверу вообще, а прямо отдаёт статические данные из `web/src/agents.ts`. Существует, чтобы экраны раздела «Агенты» уже сейчас были написаны против стабильного API-слоя, а не напрямую против моков.

## Ключевое

- `listAgents(): Promise<Agent[]>` — список агентов (мок из `AGENTS`).
- `agentQueue(): Promise<PendingAction[]>` — очередь действий, ожидающих подтверждения (мок `QUEUE`).
- `agentLog(): Promise<AgentLogEntry[]>` — журнал выполненных агентами действий (мок `AGENT_LOG`).
- Реэкспорт `AGENTS`, `QUEUE`, `AGENT_LOG` — для мест, которым нужны сырые сиды, а не промис.

## Как это работает

Каждая функция просто оборачивает статический массив в `Promise.resolve(...)`, имитируя асинхронный сетевой вызов. Комментарий в начале файла явно фиксирует контракт: когда на бэкенде появятся эндпоинты `/api/v1/agents/*`, тело этих функций заменится на `apiFetch(...)` + `withFallback` (как в остальных модулях `api/`), а сигнатуры (`Promise<Agent[]>` и т.д.) останутся теми же — экраны менять не придётся.

## Связи

Импортирует всё из `web/src/agents.ts` (типы `Agent`, `PendingAction`, `AgentLogEntry` и сами массивы). Реэкспортируется через `web/src/api/index.ts` как пространство имён `agentsApi`. Потребляется экранами раздела «Агенты» (`web/src/pages/agents`).

## На что обратить внимание

В отличие от всех остальных файлов `api/`, здесь нет `apiFetch`/`withFallback`/`ApiError` — это единственный модуль API-слоя, целиком состоящий из заглушек, и это осознанная промежуточная стадия, а не недосмотр.
