# web/src/api/terminal.ts

Обёртка над единственным эндпоинтом `POST /api/v1/terminal/exec` — «командной строкой администратора» NEX: текстовая команда уходит на бэкенд, а обратно приходит структурированный результат (текст, таблица или KPI-плитки), который фронтенд только рисует, не интерпретируя саму команду.

## Ключевое

- `interface TermKPI` — одна KPI-плитка результата (`label`, `value`).
- `interface TermResult` — унифицированный результат выполнения команды: `kind` (`'text' | 'table' | 'kpi'`) плюс соответствующие поля (`text`, `columns`/`rows` для таблицы, `kpis`), опциональные `title` и `hint`.
- `terminalExec(line): Promise<TermResult>` — отправляет строку команды на бэкенд и возвращает структурированный результат; требует права `terminal:exec` (админ) на сервере.
- `TERMINAL_BACKEND_TOKENS: Set<string>` — множество первых слов команд (на русском и английском), которые понимает бэкенд-модуль terminal — зеркало его alias-карты на сервере.

## Как это работает

В режиме с настроенным бэкендом (`VITE_API_URL` задан) фронтенд смотрит на первое слово введённой команды: если оно есть в `TERMINAL_BACKEND_TOKENS`, строка целиком уходит на сервер через `terminalExec`, и данные в ответе настоящие (из БД). В демо-режиме (`VITE_API_URL` пуст) `terminalExec` вообще не вызывается — те же самые команды исполняет локальный движок-эмулятор прямо в `web/src/pages/Home.tsx`, который знает более широкий набор команд для демонстрации.

## Связи

Зависит только от `apiFetch` из `./client`. Используется терминальным виджетом (командная строка) в `web/src/pages/Home.tsx`, который сверяется с `TERMINAL_BACKEND_TOKENS`, чтобы решить — слать команду на сервер или обработать её локально.

## На что обратить внимание

`TERMINAL_BACKEND_TOKENS` — это список, который нужно вручную поддерживать синхронно с реальной alias-картой бэкенд-модуля `internal/module/terminal`: если на сервере добавят новый алиас команды, а сюда не добавят соответствующее слово, команда в API-режиме будет ошибочно уходить в локальный демо-движок вместо настоящего бэкенда (или наоборот, в зависимости от логики выбора на странице).
