# web/src/api/campus.ts

Обёртка над `/api/v1/campus/*` — данные учебного модуля «Кампус»: группы, студенты, учебный журнал (оценки). Обслуживает экраны «Люди → Студенты/Группы» и учебный журнал, приводя DTO бэкенда к формам, которые ожидает UI-прототип (`Group`/`Student` из `../data`).

## Ключевое

- `interface ApiGroup`, `ApiStudent`, `ApiGrade` — DTO бэкенда (`internal/module/campus/http.go`): группа (код, название, число активных студентов), студент (ФИО одной строкой, статус, привязка к группе), строка журнала оценок.
- `STATUS_LABEL` — маппинг статуса студента бэкенда (`active`/`academic`/`expelled`/`graduated`) в русскую метку прототипа.
- `groupFromApi`, `studentFromApi` — функции-адаптеры DTO → форма прототипа; `studentFromApi` разбирает `full_name` по пробелам на фамилию/имя/отчество (наивно, позиционно).
- `listGroups(): Promise<Group[]>` — список групп через `withFallback`, при недоступности бэкенда — сид `seedGroups`.
- `listStudents(): Promise<Student[]>` — список студентов через `withFallback`, фолбэк — `seedStudents`.
- `listJournal(params): Promise<ApiGrade[]>` — оценки с опциональными query-параметрами (`group`, `student`, `subject`); фолбэк — пустой массив (нет мок-данных журнала).

## Как это работает

И `listGroups`, и `listStudents` идут по общему паттерну `withFallback`: пробуют реальный запрос через `apiFetch`, при успехе — трансформируют массив DTO в массив прототипной формы через `.map(...FromApi)`, при любой ошибке (бэкенд не настроен, сеть недоступна) — тихо возвращают локальный сид. `groupFromApi`/`studentFromApi` присваивают числовой `id` по позиции в массиве (`i + 1`), поскольку прототипные типы используют числовой id, а бэкенд отдаёт UUID-строки — это осознанное упрощение под текущий UI, не сквозной идентификатор.

## Связи

Импортирует `groups`/`students`/типы `Group`/`Student` из `../data` (сиды и форма прототипа) и `apiFetch`/`withFallback` из `./client`. Реэкспортируется через `web/src/api/index.ts` как пространство имён `campusApi`. Потребляется экранами `web/src/pages/people` (Students, Groups) и `web/src/pages/academic` (Journal).

## На что обратить внимание

Присвоение `id: i + 1` в адаптерах означает, что числовой id элемента не стабилен между запросами и не совпадает с настоящим UUID сервера — код, который где-то полагается на этот id как на постоянный идентификатор сущности (например, кэширует его или сравнивает между перезагрузками списка), рискует получить рассинхронизацию.
