# web/src/beta/store.ts

Локальный «фейковый backend» для бета-функций проекта: пока на Go-сервере ещё нет соответствующих модулей, данные этих разделов живут в `localStorage` браузера, но по форме API (CRUD + массовые операции + история изменений) специально совпадают с тем, как будет выглядеть настоящий серверный API. Смысл в том, что когда backend появится, заменить нужно будет только этот файл (на обёртки над HTTP-запросами) — компоненты, которые используют `useCollection`, менять не придётся.

## Ключевое

- `Entity` — базовый интерфейс сущности: `id`, `createdAt`, `updatedAt`.
- `Collection<T>` — тип коллекции с операциями `add`, `update`, `remove`, `removeMany`, `patchMany`, `replace`, `reset`.
- `useCollection<T>(key, seed)` — главный хук: даёт реактивную коллекцию поверх `localStorage` под ключом `nex-beta:<key>`; `seed` — начальные данные, используются только если хранилище ещё пусто.
- `useLocalValue<T>(key, initial)` — хук для одиночного значения (не списка) в localStorage, например настроек отображения; возвращает `[value, setValue]`, как `useState`.
- `uid(prefix)` — генерирует короткий более-менее уникальный id вида `prefix_<время36><случайные символы>`.
- `nowIso()` — текущее время в ISO-формате.
- `humanTime(iso)` — форматирует ISO-дату в читаемую строку («сегодня, 14:32» или «12 июл, 14:32»).

## Как это работает

Данные читаются/пишутся через `read`/`write` — обёртки над `JSON.parse`/`JSON.stringify` и `localStorage`, с защитой от битого JSON и переполнения квоты (тихо игнорируется). Каждый вызов `useCollection` с одним и тем же `key` подписывается на общий реестр слушателей (`listeners`), поэтому если в другом компоненте (или другой вкладке через событие `storage`) данные под этим ключом изменились, все использующие этот ключ компоненты автоматически перечитывают и обновляют состояние — это имитирует «живые» обновления, как если бы было real-time соединение с сервером. Все мутирующие методы (`add`, `update`, `remove`...) — это `useCallback`, которые читают актуальные данные из localStorage (не из React state, чтобы избежать гонок), применяют изменение и вызывают `commit`, который пишет в localStorage, обновляет локальный `state` и уведомляет остальных слушателей.

## Связи

Не зависит от других модулей проекта (только React). Является фундаментом для `web/src/beta/manager.tsx` (использует `Collection`, `Entity`, `uid`) и для всех бета-экранов `web/src/pages/*-beta.tsx`, которые вызывают `useCollection` для хранения своих данных (студенты, финансы, задачи и т.п. — до переноса на реальный backend).

## На что обратить внимание

Ключи в localStorage читаются напрямую в момент мутации (`read<T>(key, seed)` внутри `add`/`update`/...), а не берутся из React-состояния — это намеренно, чтобы несколько независимых экземпляров хука с одним `key` не затирали изменения друг друга при почти одновременных обновлениях.
