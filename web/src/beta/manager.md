# web/src/beta/manager.tsx

Универсальный компонент-«конструктор» экрана управления сущностями: таблица со списком, поиск, создание/редактирование через модалку, массовое удаление, экспорт в CSV и импорт из JSON. Он существует, чтобы разделы вроде финансов, учебного процесса, кадров и безопасности не писали одну и ту же логику CRUD-таблицы заново — вместо этого им достаточно описать колонки и поля формы, а `EntityManager` соберёт весь экран.

## Ключевое

- `EntityManager<T>` — главный экспортируемый компонент: принимает `Collection<T>` (из `store.ts`), список колонок таблицы, список полей формы и рисует полноценный CRUD-экран.
- `Editor<T>` — внутренний (неэкспортируемый) компонент модального редактора создания/изменения записи, рендерит поля формы по описанию `FieldDef<T>[]`.
- `Col<T>` — тип описания колонки таблицы (ключ поля, подпись, вид отображения: деньги/статус/чип/текст).
- `FieldDef<T>` — тип описания поля формы редактора (ключ, подпись, тип ввода, варианты для select/multi-select).
- `rub(n)` — форматирует число в рублёвую строку вида «₽ 12 345».
- `statusTone(s)` — по тексту статуса (например «оплачено», «просрочено», «черновик») подбирает CSS-класс цвета чипа (успех/опасность/предупреждение/нейтральный) через регулярки по русским словам.

## Как это работает

Компонент держит локальное состояние поиска, выбранных строк, редактируемой записи и подтверждения удаления. Список строк фильтруется по подстроке во всех отображаемых колонках (`useMemo`). Экспорт CSV собирает строки из уже отфильтрованного списка и скачивает Blob; импорт читает JSON-файл, ожидает массив объектов и добавляет их через `col.add`, присваивая новый `id`. Отрисовка ячейки (`cell`) зависит от `kind` колонки: деньги форматируются через `rub`, статусы — через `Chip` с цветом из `statusTone`, массивы — через `join(', ')`. Все мутации данных идут через методы `Collection` (`add`, `update`, `removeMany`), которые в конечном счёте пишут в localStorage.

## Связи

Зависит от `web/src/ui.tsx` (`useApp` для тостов, `Chip` для статусов), `web/src/beta/store.ts` (типы `Entity`, `Collection`, функция `uid`) и `web/src/beta/kit.tsx` (все UI-примитивы: `Modal`, `Field`, `Text`, `Select`, `Area`, `MultiSelect`, `AddButton`, `Confirm`, `Empty`, `RowCheck`). Используется бета-версиями экранов (`web/src/pages/*-beta.tsx`) как основной строительный блок для разделов со списками сущностей.
