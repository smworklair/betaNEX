# web/src/beta/kit.tsx

Библиотека мелких переиспользуемых UI-компонентов для «бета»-разделов приложения (тех, что ещё работают на localStorage, а не на реальном backend API). Задача файла — чтобы модальные окна, формы, теги и подтверждения выглядели и вели себя одинаково во всех бета-экранах, вместо того чтобы каждый экран рисовал их заново. Все визуальные стили вынесены в `index.css` под классами `.bk-*`, сам файл отвечает только за разметку и поведение.

## Ключевое

- `Modal` — базовое модальное окно с оверлеем, закрытием по Escape/клику мимо и блокировкой скролла страницы, пока открыто.
- `Field` — обёртка «подпись + поле + подсказка» для форм.
- `Text`, `Area`, `Select` — простые контролируемые обёртки над `<input>`, `<textarea>`, `<select>`.
- `TagInput` — ввод списка тегов: Enter/запятая добавляет тег, Backspace на пустом поле удаляет последний.
- `MultiSelect` — выпадающий мультивыбор (например, исполнители задачи), закрывается по клику вне компонента.
- `Toolbar` — верхняя панель раздела: поле поиска слева, произвольные действия справа.
- `AddButton` — единая кнопка «Создать» с иконкой плюса.
- `BulkBar` — панель массовых действий, появляется только когда есть выбранные строки (`count > 0`).
- `Confirm` — модалка подтверждения (обычно для удаления), построена поверх `Modal`.
- `Empty` — заглушка «пусто» с иконкой, текстом и опциональным действием.
- `RowCheck` — чекбокс строки таблицы (кастомная кнопка вместо `<input type=checkbox>`).
- `SectionNote` — небольшая текстовая пометка (например «в разработке»).

## Как это работает

Компоненты полностью «контролируемые» — не хранят состояние сами (кроме локального UI-состояния типа открыт/закрыт `MultiSelect`), а получают `value`/`onChange` от родителя. Это обычный паттерн для набора UI-примитивов: каждый компонент решает одну маленькую задачу и не знает ничего о бизнес-логике или о том, где хранятся данные.

## Связи

Не зависит от других модулей проекта, кроме React и `lucide-react` (иконки). Активно используется в `web/src/beta/manager.tsx` (там строятся формы CRUD-редактора и панель инструментов таблицы) и напрямую в бета-версиях экранов (`*-beta.tsx`), где нужны модалки, теги или мультивыбор вне универсального менеджера.
