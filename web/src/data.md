# web/src/data.ts

Центральный файл с моковыми (тестовыми) данными для всего прототипа NEX: студенты, группы, преподаватели, финансы, расписание, аудит безопасности, почта, чаты, соцлента задач и уведомлений. Как указано в комментарии в начале файла — в продакшене все эти данные будут приходить с backend (ядро + модули), а здесь это локальный seed и «мок-телеметрия», форма которой заранее повторяет будущие ответы API. Это самый большой источник тестовых данных в проекте, на нём построена демонстрация практически всех экранов.

## Ключевое

- `Role`, `roleLabel` — роли пользователей системы (admin/teacher/accountant/student) и их подписи на русском.
- `Group`, `groups` / `Student`, `students` / `StaffMember`, `staff` — базовые сущности учебного процесса: группы, студенты, сотрудники.
- `subjectsByGroup`, `gradesFor(groupName)` — предметы по группам и детерминированная генерация псевдооценок (стабильных между рендерами за счёт формулы по `id`).
- `sessions`, `auditEvents`, `criticalActions`, `failedLogins`, `failedLoginTrend` — телеметрия безопасности: активные сессии, журнал аудита, неудачные входы.
- `notifications`, `aiInsights`, `services` — системные уведомления, ИИ-инсайты и статус здоровья сервисов (для дашборда/командного центра).
- `finance` (`payments`, `scholarships`), `charges`, `payroll`, `budgetLines`, `reports` — данные для бухгалтерии/финансового блока.
- `admissions` — заявки приёмной комиссии.
- `scheduleDays`, `scheduleSlots`, `calEvents`, `exams` — расписание занятий, календарь событий, экзамены.
- `posts`, `postComments` — записи и комментарии внутренней соцленты.
- `chats`, `thread`, `threads` — мессенджер: список чатов и переписки по каждому.
- `emails`, `MAIL_FOLDERS` — данные встроенного почтового клиента (папки, письма, вложения).
- `tasks`, `nexLog` — задачи пользователя и журнал действий ИИ-агента NEX.

## Как это работает

Файл не содержит логики — это просто набор экспортируемых констант и TypeScript-интерфейсов, описывающих их форму. Единственная нетривиальная функция — `gradesFor`, которая генерирует «случайные», но воспроизводимые оценки по формуле от `id` студента и номера столбца, чтобы журнал оценок выглядел одинаково при каждом рендере вместо использования `Math.random()`.

## Связи

Не имеет собственных зависимостей. Импортируется огромным числом файлов проекта, которым нужны тестовые данные до появления реального backend — среди файлов из этого задания на него явно опирается `web/src/blocks.tsx` (`students`, `finance`, `gradesFor`); также ожидаемо используется большинством экранов в `web/src/pages/*` (не-бета версии) и компонентами дашборда/аудита/финансов/почты/чатов.
