# web/src/md.tsx

Собственный минималистичный markdown-рендерер без внешних зависимостей, специально для ответов ИИ-помощника NEX. Поддерживает базовый набор разметки, которую генерирует LLM: жирный/курсив текст, инлайн-код, маркированные и нумерованные списки, заголовки, горизонтальные линии. Существует, чтобы не подключать тяжёлую markdown-библиотеку ради простого форматирования чатовых сообщений.

## Ключевое

- `Md({ text })` — единственный экспортируемый компонент: принимает сырой markdown-текст и возвращает React-разметку.
- `inline(text)` — внутренняя функция, разбирает инлайн-разметку внутри одной строки/абзаца: `**жирный**`, `` `код` ``, `*курсив*`/`_курсив_`.
- `isBullet`, `isNum`, `isHead`, `isHr` — внутренние хелперы-регэкспы, определяющие тип строки (маркированный пункт, нумерованный пункт, заголовок `#`-`####`, горизонтальная линия).

## Как это работает

`Md` построчно проходит текст и группирует соседние строки одного типа в блоки: последовательные `-`/`*`/`•`-строки — в `<ul>`, последовательные `1.`/`2)`-строки — в `<ol>`, строки `#`...`####` — в заголовок, остальное — в абзацы `<p>`, которые сшиваются до следующей пустой строки или начала другого блока. Внутри каждого блока текст ещё раз прогоняется через `inline()`, которая ищет жирный/курсив/код через один общий regex и заменяет совпадения на соответствующие React-элементы, сохраняя порядок обычного текста между ними.

## Связи

Не зависит от других файлов проекта (только React). Используется везде, где рендерится текст ответа NEX — ожидаемо в `web/src/pages/Chat.tsx`, `web/src/pages/Home.tsx` (терминал) и `web/src/terminal.tsx`.
