# web/src/dock.tsx

Каталоги пунктов навигации для верхней панели (десктоп) и нижнего докбара (мобильная версия). Файл существует, чтобы описать, какие разделы вообще доступны для размещения в навигации и какие из них показаны по умолчанию — а пользователь может часть кнопок скрыть или переставить, и этот выбор сохраняется в настройках (prefs) в localStorage.

## Ключевое

- `TopItem`, `TOPBAR_CATALOG` — полный список пунктов верхней панели (десктоп): id, подпись, иконка. `id` совпадает с id раздела в `SECTIONS` (определено в другом месте приложения).
- `DEFAULT_TOPBAR` — id всех пунктов верхней панели по умолчанию (все включены).
- `TOPBAR_MIN` — минимальное число пунктов, которое нельзя убрать из верхней панели.
- `DockItem`, `DOCK_CATALOG` — полный список кнопок нижнего докбара (мобайл): в отличие от `TopItem`, у каждого элемента есть `page` — конкретная страница, куда ведёт кнопка (например у `mail` id и page различаются — это отдельная кнопка мессенджера).
- `DOCK_BY_ID` — тот же каталог доступа, но в виде словаря по id для быстрого поиска.
- `DEFAULT_DOCK` — 5 кнопок дока по умолчанию, включая мессенджер.
- `DOCK_MIN`, `DOCK_MAX` — ограничения на количество кнопок в доке (от 2 до 6).

## Как это работает

Это чисто декларативный файл-каталог без логики — константы и типы. Реальная логика показа/скрытия/перестановки пунктов навигации, а также чтение/запись выбора пользователя из/в localStorage находится в других местах приложения (компоненты навигации и хранилище настроек `prefs`), этот файл им лишь поставляет список доступных вариантов и значения по умолчанию.

## Связи

Зависит от иконок `lucide-react`. Используется компонентами навигации приложения (верхняя панель и мобильный докбар — вероятно в `App.tsx` или отдельном компоненте навигации, не входящем в текущий список файлов) для построения меню и для экрана настроек, где пользователь включает/выключает и переставляет пункты.
