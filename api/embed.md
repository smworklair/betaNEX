# api/embed.go

Крошечный файл, который встраивает файл `openapi.yaml` прямо в бинарник `nexd` на этапе компиляции (через директиву `go:embed`). Это позволяет серверу отдавать актуальную версию контракта API, не читая файлы с диска и не полагаясь на то, что рядом с бинарником лежит спека.

## Ключевое

- `OpenAPI []byte` — байтовое содержимое файла `api/openapi.yaml`, встроенное в бинарник на этапе сборки директивой `//go:embed openapi.yaml`.

## Как это работает

При компиляции Go читает `openapi.yaml` из той же директории и кладёт его содержимое в переменную `OpenAPI`. Дальше это просто срез байт в памяти — никакого чтения файловой системы в рантайме не происходит. Пакет называется `api` и физически лежит в `api/`, рядом с самим yaml-файлом.

## Связи

- Зависит только от файла `api/openapi.yaml`, который лежит рядом.
- Используется в `cmd/nexd/main.go`: значение `api.OpenAPI` передаётся в `httpapi.NewRouter` как `RouterConfig.OpenAPI`, чтобы сервер мог отдавать спеку по `GET /api/v1/openapi.yaml`.

## На что обратить внимание

Так как контракт встраивается в бинарник во время сборки, обновление `openapi.yaml` требует пересборки `nexd` — простой замены yaml-файла на диске недостаточно.
